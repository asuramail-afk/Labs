Container Queries

中級

container-type: inline-size を指定した要素を「コンテナ」として宣言し、@container ルールでそのコンテナ幅に応じたスタイルを記述できます。 ビューポート幅ではなくコンポーネント自身のサイズに依存するため、再利用性の高いコンポーネント設計が可能です。

▶ 右下角をドラッグして幅を変えてみよう (resizable)
Container Query Card
幅が400pxを超えると横並びレイアウトに自動切替。メディアクエリ不要。
/* コンテナを宣言 */
.card-wrapper {
  container-type: inline-size;
  container-name: card-container;
}

/* コンテナ幅でスタイル切替 */
@container card-container (inline-size > 400px) {
  .card {
    flex-direction: row;
  }
}

/* cqi/cqb 単位も使える */
.card-title {
  font-size: 4cqi; /* コンテナ幅の4% */
}

Subgrid

中級

grid-template-columns: subgrid を子グリッドに指定すると、親グリッドのトラック定義を引き継ぎます。 カードコンポーネントを並べたとき、タイトル・説明文・ボタン行の縦位置が自動で揃います。

▶ 各カードの説明文の長さが違っても下端ボタン行が揃う
Card A
短い説明。
詳細 →
Card B
少し長めの説明テキストがここに入ります。行数が増えても下端は揃います。
詳細 →
Card C
中程度の説明テキスト。Subgridが整列を維持します。
詳細 →
/* 親グリッドで行定義 */
.grid-wrapper {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  grid-template-rows: auto 1fr auto;
  gap: 12px;
}

/* 子グリッドが親トラックを継承 */
.card {
  display: grid;
  grid-template-columns: subgrid;
  grid-template-rows: subgrid;
  grid-row: span 3;
}

Anchor Positioning

上級

anchor-name で要素に名前を付け、position-anchor と anchor() 関数でツールチップやポップオーバーをアンカー要素に紐付けて自動配置できます。 Chrome 125+ 実装済み、Firefox は未実装（2026年時点）のため、フォールバック設計が必要です。

▶ アンカーボタンの直下にツールチップが配置される (Chrome 125+)
ここがツールチップ (anchor positioning)
/* アンカー要素に名前を付ける */
.anchor-btn {
  anchor-name: --my-anchor;
}

/* ツールチップをアンカーに紐付け */
.tooltip {
  position: absolute;
  position-anchor: --my-anchor;
  top: anchor(bottom); /* アンカー下辺に合わせる */
  left: anchor(center);
  translate: -50% 0;
}

/* @supports でフォールバック */
@supports not (anchor-name: --x) {
  .tooltip { top: calc(100% + 8px); }
}

Aspect Ratio

基本

aspect-ratio: 16/9 で縦横比を1行指定。height を省略しても幅から高さが自動計算されます。 レスポンシブな画像・動画プレースホルダー・カードサムネイルの定番プロパティです。

▶ widthだけ指定、heightはaspect-ratioから自動計算
16/9
1/1
/* 16:9 の動画サムネイル */
.video-thumb {
  aspect-ratio: 16/9;
  width: 100%;
  /* height 不要: 幅から自動計算 */
}

/* 正方形アバター */
.avatar {
  aspect-ratio: 1/1;
  width: 48px;
}

/* auto キーワード: 画像本来の比率を維持 */
img {
  aspect-ratio: auto 16/9;
}

:has()

中級

CSS初の「親セレクタ」。form:has(input:invalid) で「無効な入力を含むフォーム」全体をスタイルできます。 2024 Baseline 達成済みで全モダンブラウザ対応。JS不要のバリデーションUIが実現できます。

▶ メールアドレスを入力してフォーム全体の色が変化するか確認
正しいメールアドレスを入力してください 
/* 無効な入力を含むフォームを赤枠に */
form:has(input:invalid) {
  border-color: red;
  background: rgba(203,36,49,.04);
}

/* 有効な入力を含むフォームを緑枠に */
form:has(input:valid:not(:placeholder-shown)) {
  border-color: green;
}

/* 直下の子要素のみ対象 */
form:has(> input:invalid) {
  outline: 2px solid red;
}

:is() / :where()

中級

:is() と :where() はどちらもセレクタリストを引数に取り、繰り返しセレクタを短縮できます。 違いは詳細度：:is() は引数中の最大詳細度を引き継ぎますが、:where() は常に詳細度 0 です。

▶ :is() で見出し色を一括指定、:where() で詳細度0のリセット

h4 見出し (:is() で青色)

段落テキスト (:where() で文字色指定)

h5 見出し (:is() で青色)

もう一つの段落テキスト

セレクタ 詳細度 上書きのしやすさ 用途
:is(h1, .title) 0,1,0 (class分) 低め 通常スタイル付け
:where(h1, .title) 0,0,0 (常に0) 高い リセット・ベーススタイル
h1, h2, h3 {...} 0,0,1 (要素分) 通常 従来の書き方 
/* 繰り返しセレクタを短縮 */
/* Before: */
.nav a:hover, .footer a:hover, .sidebar a:hover { color: blue; }

/* After: :is() で詳細度あり */
:is(.nav, .footer, .sidebar) a:hover { color: blue; }

/* After: :where() で詳細度0（上書き容易） */
:where(.nav, .footer, .sidebar) a:hover { color: blue; }

/* ネスト見出しに一括でスタイル */
.article :is(h1, h2, h3) {
  font-weight: 700;
  line-height: 1.3;
}

:focus-visible & :focus-within

基本

キーボード操作時だけフォーカスリングを表示し、マウスクリック時は非表示にする。:focus-within は子孫にフォーカスがある親にも一致する。

▍ LIVE DEMO

← Tabキーで移動 → リング表示 / マウスクリック → リングなし

← フォーム内にフォーカスがある間、外枠が緑色になる

/* キーボード時だけリング */
.btn:focus-visible {
  outline: 3px solid #89b4fa;
  outline-offset: 3px;
}

/* 子孫にフォーカスがある親 */
form:focus-within {
  border-color: #a6e3a1;
  background: #1a2b1a;
}

@layer（カスケードレイヤー）

中級

カスケードレイヤーで優先順位を明示管理。レイヤー順が最優先され、詳細度はレイヤー内でのみ比較される。

▍ LIVE DEMO
util層(緑・bold) が reset層(赤) より勝つ → 詳細度が同じでもレイヤー順で決まる
レイヤー色指定結果
reset（1番目）color: red(#f38ba8)負け
base（2番目）color: blue(#89b4fa)負け
util（3番目）color: green(#a6e3a1)勝ち
un-layered全レイヤーより優先（最強）
@layer reset, base, util; /* 順序宣言 */

@layer reset {
  .box { color: red; }
}
@layer base {
  .box { color: blue; }
}
@layer util {
  .box { color: green; } /* 勝ち */
}
/* un-layered は全レイヤーより優先 */

上級

▍ LIVE DEMO

scope内のh2（紫スタイル適用）

  }
}
/* .card 外の h2 は無影響 */

/* 下限指定: .card から .footer の手前まで */
@scope (.card) to (.footer) { }

@property（型付きカスタムプロパティ）

中級

カスタムプロパティに型・初期値・継承制御を付与。transitionが効くようになる（素の var() は transition 不可）。

▍ LIVE DEMO
@property あり → スムーズな色遷移
@property なし（syntax:'*'）→ 突然切り替わる 
@property --hue {
  syntax: '<number>';
  inherits: false;
  initial-value: 200;
}

.swatch {
  background: hsl(var(--hue), 70%, 55%);
  transition: --hue 0.6s ease; /* 型付きなので効く */
}

/* JS: el.style.setProperty('--hue', 30) → 滑らか遷移 */

Scroll-driven Animations

上級

animation-timeline: scroll() と view() でJSなしにスクロール連動アニメーション。

Chrome 115+ のみ対応。Firefox は 2026 時点でも未対応（アニメーション静止）。

animation-timeline: scroll() — ページスクロール連動プログレスバー

animation-timeline: view() — ビュー進入アニメーション

Item A — ビュー内に入るとフェードイン
Item B — ビュー内に入るとフェードイン
Item C — ビュー内に入るとフェードイン
/* scroll(): スクロール量をアニメ進捗に */
@keyframes grow-bar { from { width: 0% } to { width: 100% } }

.progress-bar {
  animation: grow-bar linear;
  animation-timeline: scroll(root);
  animation-range: 0% 100%;
}

/* view(): 要素のビュー進入に連動 */
@keyframes fade-in-up {
  entry 0%  { opacity: 0; transform: translateY(40px); }
  entry 100% { opacity: 1; transform: translateY(0); }
}

.item {
  animation: fade-in-up linear;
  animation-timeline: view();
  animation-range: entry 0% entry 100%;
}
