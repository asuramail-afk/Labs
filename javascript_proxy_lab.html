00

Proxyとは何か — 概念を掴む

基本

Proxy はオブジェクトの前に立つ「門番」です。プロパティの読み書き・削除・関数呼び出しなど、オブジェクトへのあらゆる操作を途中で横取り（インターセプト）し、独自のロジックを差し込めます。

🔰 たとえ話で理解する
ホテルのフロントデスクを想像してください。宿泊客（コード）が部屋（オブジェクト）に何か頼むとき、必ずフロント（Proxy）を経由します。フロントは「荷物を届ける」「部屋を掃除する」「チェックアウト」どれも一旦受け取り、ルールに従って処理します。 
  あなたのコード        Proxy（門番）        ターゲットオブジェクト
 ┌──────────┐      ┌─────────────┐      ┌──────────────┐
 │ obj.name  │─────▶│ get トラップ  │─────▶│ { name: "太郎" } │
 │ obj.age=25│─────▶│ set トラップ  │─────▶│ { age: 25 }    │
 │ delete obj│─────▶│ delete トラップ│─────▶│ (プロパティ削除) │
 └──────────┘      └─────────────┘      └──────────────┘
Proxy の 3つの登場人物
// ① ターゲット（target）: 元のオブジェクト
const target = { name: "太郎", age: 30 };

// ② ハンドラ（handler）: トラップ関数を集めたオブジェクト
const handler = {
  get(target, prop) {
    console.log(`"${prop}" が読まれました`);
    return target[prop];
  }
};

// ③ プロキシ（proxy）: target + handler を組み合わせた新オブジェクト
const proxy = new Proxy(target, handler);

proxy.name; // → "name が読まれました" → "太郎"
用語役割たとえ
target元のオブジェクトホテルの部屋
handlerトラップ関数をまとめたオブジェクトフロントの業務マニュアル
trap個々の操作をフックする関数「荷物配達」「掃除」等の各業務手順
proxytarget + handler の合体物フロント経由でしか触れない部屋
01

最初のProxy作成 — 空ハンドラ

基本

ハンドラに何もトラップを定義しない場合、Proxyは完全に透過的（パススルー）に動作します。つまり、元のオブジェクトとまったく同じ振る舞いをします。これが出発点です。

空ハンドラ — 透過Proxy
const target = { x: 1, y: 2 };
const proxy = new Proxy(target, {}); // handler は空オブジェクト

console.log(proxy.x);      // 1   ← そのまま通過
proxy.z = 3;
console.log(target.z);     // 3   ← target にも反映される！
console.log(proxy === target); // false ← 別オブジェクト

▶ 実行してみよう

🔰 proxy と target は別モノ
proxy === target は false です。Proxy は target を「ラップ」した新しいオブジェクト。しかし proxy への操作は target に反映されます（共有参照）。
02

getトラップ — 読み取りの横取り

基本

get(target, property, receiver) はプロパティが読み取られるたびに呼ばれます。最も基本的で最も頻出するトラップです。

get トラップの基本
const user = { name: "花子", age: 25 };

const proxy = new Proxy(user, {
  get(target, prop, receiver) {
    console.log(`[GET] ${prop}`);

    // 存在しないプロパティにはデフォルトメッセージ
    if (!(prop in target)) {
      return `"${prop}" は存在しません`;
    }
    return target[prop];
  }
});

console.log(proxy.name);    // [GET] name → "花子"
console.log(proxy.email);   // [GET] email → "email は存在しません"

▶ 実行してみよう

引数意味例
target元のオブジェクトuser
propアクセスされたプロパティ名"name"
receiverproxy自身（or 継承先）通常は proxy
03

setトラップ — 書き込みの横取り

基本

set(target, property, value, receiver) はプロパティへの代入時に呼ばれます。バリデーション（型チェック、範囲チェック等）を入れられる最も実用的なトラップです。必ず true を返すことが重要です。

set トラップで型チェック
const person = { name: "太郎", age: 30 };

const proxy = new Proxy(person, {
  set(target, prop, value, receiver) {
    if (prop === "age") {
      if (typeof value !== "number") {
        throw new TypeError("age は数値のみ");
      }
      if (value < 0 || value > 150) {
        throw new RangeError("age は 0〜150");
      }
    }
    target[prop] = value;
    return true; // ← 必須！
  }
});

proxy.age = 25;       // OK
// proxy.age = "二十五"; // → TypeError!
// proxy.age = -5;      // → RangeError!

04

has / deleteProperty トラップ

中級

has は in 演算子を、deleteProperty は delete 演算子をインターセプトします。プロパティの存在チェックや削除を制御できます。

has — プロパティ隠蔽
const secrets = { password: "abc123", name: "太郎" };

const proxy = new Proxy(secrets, {
  has(target, prop) {
    if (prop === "password") {
      return false; // 存在を隠す
    }
    return prop in target;
  }
});

console.log("name" in proxy);     // true
console.log("password" in proxy); // false (隠蔽！)
console.log(proxy.password);       // "abc123" (get は制限していないので取れる)
deleteProperty — 削除の制限
const config = { host: "localhost", port: 3000, debug: true };

const proxy = new Proxy(config, {
  deleteProperty(target, prop) {
    if (prop === "host" || prop === "port") {
      throw new Error(`"${prop}" は削除禁止です`);
    }
    delete target[prop];
    return true;
  }
});

delete proxy.debug; // OK — debug は削除可能
// delete proxy.host; // → Error: "host" は削除禁止です

05

applyトラップ — 関数呼び出しの横取り

中級

apply(target, thisArg, args) は関数が呼び出されたときに発火します。target が関数の場合のみ使用可能です。ログ記録、キャッシュ、引数変換などに活用できます。

apply — 実行時間の計測ラッパー
function heavyCalc(n) {
  let sum = 0;
  for (let i = 0; i < n; i++) sum += i;
  return sum;
}

const timedCalc = new Proxy(heavyCalc, {
  apply(target, thisArg, args) {
    const start = performance.now();
    const result = target.apply(thisArg, args);
    const ms = (performance.now() - start).toFixed(2);
    console.log(`実行時間: ${ms}ms`);
    return result;
  }
});

timedCalc(1000000); // → 実行時間: X.XXms → 499999500000

▶ 実行してみよう

🔰 apply は「関数専用」
普通のオブジェクト（ { } ）に apply トラップを仕掛けてもエラーになります。target が関数（typeof target === "function"）のときだけ使えます。
06

constructトラップ — new操作の横取り

中級

construct(target, args, newTarget) は new 演算子によるインスタンス生成をインターセプトします。必ずオブジェクトを返す必要があります。

construct — シングルトン風パターン
function Database(name) {
  this.name = name;
  this.createdAt = new Date().toISOString();
}

let instance = null;

const SingletonDB = new Proxy(Database, {
  construct(target, args, newTarget) {
    if (!instance) {
      instance = new target(...args);
      console.log("新規インスタンス作成");
    } else {
      console.log("既存インスタンスを再利用");
    }
    return instance;
  }
});

const db1 = new SingletonDB("myDB");
const db2 = new SingletonDB("otherDB");
console.log(db1 === db2); // true

バリデーション実装 — 実践パターン

中級

複数のプロパティに対して型・範囲・形式のバリデーションを一括管理する実用パターンです。スキーマ定義とProxyを組み合わせます。

スキーマ駆動バリデーション
const schema = {
  name:  { type: "string",  minLen: 1, maxLen: 50 },
  email: { type: "string",  pattern: /^[^\s@]+@[^\s@]+\.[^\s@]+$/ },
  age:   { type: "number",  min: 0, max: 150 }
};

function createValidated(data, schema) {
  return new Proxy(data, {
    set(target, prop, value) {
      const rule = schema[prop];
      if (!rule) { target[prop] = value; return true; }

      if (typeof value !== rule.type) {
        throw new TypeError(`${prop}: ${rule.type}型が必要`);
      }
      if (rule.minLen && value.length < rule.minLen) {
        throw new Error(`${prop}: ${rule.minLen}文字以上`);
      }
      if (rule.pattern && !rule.pattern.test(value)) {
        throw new Error(`${prop}: 形式が不正`);
      }
      if (rule.min != null && value < rule.min) {
        throw new RangeError(`${prop}: ${rule.min}以上`);
      }
      target[prop] = value;
      return true;
    }
  });
}

const user = createValidated({}, schema);
user.name = "太郎";            // ✓ OK
user.email = "taro@test.com";  // ✓ OK
user.age = 25;                  // ✓ OK
// user.email = "not-an-email"; // → Error: email: 形式が不正

デフォルト値と存在しないプロパティ

中級

存在しないプロパティにアクセスしたとき、undefined ではなくデフォルト値を返す「スマートオブジェクト」を作ります。辞書型データやi18n翻訳オブジェクトで便利です。

デフォルト値Proxy
function withDefaults(target, defaults) {
  return new Proxy(target, {
    get(obj, prop) {
      return prop in obj ? obj[prop] : defaults[prop] ?? `[${prop} 未定義]`;
    }
  });
}

const config = withDefaults(
  { host: "production.example.com" },
  { host: "localhost", port: 3000, debug: false }
);

console.log(config.host);    // "production.example.com" (自分の値)
console.log(config.port);    // 3000 (デフォルト値)
console.log(config.debug);   // false (デフォルト値)
console.log(config.unknown); // "[unknown 未定義]"

変更監視（Observer パターン）

応用

プロパティの変更を検知してコールバックを呼ぶ「Observer」パターン。Vue.js や MobX 等のリアクティブシステムの原型です。

🔰 Vue.jsとの関係
Vue 3 の reactive() は内部で Proxy を使ってデータ変更を検知しています。この Lab で作るものは、その仕組みの簡略版です。 
  Vue 3 リアクティブシステム（簡略）

  const state = reactive({ count: 0 })
        ↓ 内部的に
  new Proxy({ count: 0 }, {
    get()  → 依存関係を追跡（track）
    set()  → 変更を通知（trigger） → DOM更新
  })
変更監視 Observable
function observable(target, onChange) {
  return new Proxy(target, {
    set(obj, prop, value) {
      const oldValue = obj[prop];
      obj[prop] = value;
      onChange({
        prop,
        oldValue,
        newValue: value,
        timestamp: new Date().toLocaleTimeString()
      });
      return true;
    },
    deleteProperty(obj, prop) {
      const oldValue = obj[prop];
      delete obj[prop];
      onChange({ prop, oldValue, newValue: undefined, action: "delete" });
      return true;
    }
  });
}

const state = observable({ count: 0, text: "hello" }, (change) => {
  console.log(`[${change.timestamp}] ${change.prop}: ${change.oldValue} → ${change.newValue}`);
});

state.count = 1;    // → [時刻] count: 0 → 1
state.text = "hi"; // → [時刻] text: hello → hi

Reflect APIとの連携

応用

Reflect は Proxy のトラップと1:1対応するメソッドを持つ組み込みオブジェクトです。トラップ内で target[prop] の代わりに Reflect.get(target, prop, receiver) を使うことで、プロトタイプチェーンや getter の this が正しく動作します。

Reflect を使うべき理由
// ❌ target[prop] — this が target に固定される
const badHandler = {
  get(target, prop) {
    return target[prop]; // getter 内の this が target になる
  }
};

// ✅ Reflect.get — receiver を渡すので this が正しい
const goodHandler = {
  get(target, prop, receiver) {
    console.log(`[GET] ${prop}`);
    return Reflect.get(target, prop, receiver);
  },
  set(target, prop, value, receiver) {
    console.log(`[SET] ${prop} = ${value}`);
    return Reflect.set(target, prop, value, receiver);
  }
};
getter で差が出る例
const base = {
  _name: "太郎",
  get greeting() {
    return `こんにちは、${this._name}です`; // ← this が重要
  }
};

const proxy = new Proxy(base, {
  get(target, prop, receiver) {
    return Reflect.get(target, prop, receiver);
  }
});

const child = Object.create(proxy);
child._name = "花子";
console.log(child.greeting); // "こんにちは、花子です" ← 正しい！
Proxy トラップ対応する Reflect メソッド
getReflect.get(target, prop, receiver)
setReflect.set(target, prop, value, receiver)
hasReflect.has(target, prop)
deletePropertyReflect.deleteProperty(target, prop)
applyReflect.apply(target, thisArg, args)
constructReflect.construct(target, args, newTarget)

Revocable Proxy — 取り消し可能

応用

Proxy.revocable() は通常の Proxy に加えて revoke() 関数を返します。一度 revoke すると、そのプロキシへのあらゆる操作が TypeError になります。API キーの無効化やセッション管理に使えます。

Revocable Proxy
const { proxy, revoke } = Proxy.revocable(
  { secret: "機密データ", public: "公開データ" },
  {
    get(target, prop, receiver) {
      console.log(`[アクセス] ${prop}`);
      return Reflect.get(target, prop, receiver);
    }
  }
);

console.log(proxy.secret); // "機密データ" — まだアクセス可能

revoke();  // ← ここでプロキシを無効化！

try {
  proxy.secret; // → TypeError: proxy が revoke 済み
} catch (e) {
  console.log(`エラー: ${e.message}`);
}

▶ 実行してみよう

🔰 使いどころ
「一定時間だけアクセスを許可する一時トークン」や「ログアウト後に即座にデータアクセスを遮断する」場合に使えます。revoke() は何度呼んでもOKで、一度無効化したProxyは復活できません。

全トラップ一覧と不変条件

応用

Proxy には全13種のトラップがあります。また、Proxy には「不変条件（invariant）」というルールがあり、ハンドラで嘘をつける範囲には制限があります。

不変条件（Invariant）の例
const obj = {};
Object.defineProperty(obj, "locked", {
  value: 42,
  writable: false,
  configurable: false
});

const proxy = new Proxy(obj, {
  get(target, prop) {
    if (prop === "locked") return 99; // 嘘の値を返そうとする
    return target[prop];
  }
});

// proxy.locked → TypeError!
// configurable:false かつ writable:false のプロパティは
// get トラップで異なる値を返せない（不変条件違反）

▶ 実行してみよう

🔰 なぜ不変条件があるのか
もしProxyがどんな嘘でもつけたら、Object.freeze() や Object.defineProperty で設定した制約が意味をなさなくなります。JavaScript の型システムの信頼性を保つため、Proxy にも「越えられない壁」が設けられています。
