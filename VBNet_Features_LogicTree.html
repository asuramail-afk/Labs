オブジェクト初期化子・コレクション初期化子・構造体・タプル・ラムダ・共有メンバー・Nullable
▼ VB.NET 主要機能
▼ 1. オブジェクト初期化子 (Object Initializer)
▼ 目的コンストラクタを増やさずプロパティ一括設定 
' 従来
Dim p As New Person()
p.Name = "Tom"
p.Age = 40

' オブジェクト初期化子
Dim p As New Person() With {
    .Name = "Tom",
    .Age = 40,
    .City = "Osaka"
}
▼ 利用シーンDTO詰め替え / LINQ / テストデータ
─DTO・エンティティへの値詰め(DB変換、API応答組立)
─LINQ Select 内で匿名型を作る場面で必須
─単体テストの入力データ準備
─ 必要性「全パラメータ版/一部版」のコンストラクタ乱立を防ぐ。プロパティが縦に並びレビュー時に読みやすい
▼ 2. コレクション初期化子 (Collection Initializer)
▼ 目的List/Dictionary を宣言と同時に中身まで初期化 
Dim names As New List(Of String) From {"Tom", "Alice", "Bob"}

Dim scores As New Dictionary(Of String, Integer) From {
    {"Math", 90},
    {"English", 75}
}
▼ 利用シーンマスタデータ / テスト期待値 / 設定値
─定数的なマスタデータ(コード値→名称)
─単体テストの期待値リスト
─設定値のハードコード
─ 必要性Add 連打よりデータが「表」として見える。要素増減時の diff も小さい
▼ 3. 構造体 (Structure)
▼ 目的軽量な「値型」のデータ塊を定義(クラスは参照型) 
Public Structure Point
    Public X As Integer
    Public Y As Integer
End Structure

Dim a As New Point With {.X = 1, .Y = 2}
Dim b = a   ' ← コピーされる(クラスなら参照共有)
b.X = 99
' a.X は 1 のまま
▼ 選ぶべきときサイズ小・不変・大量生成
─サイズが小さい(目安16バイト以下)
─不変的な値の集合(座標、色、金額+通貨)
─大量生成しGC負荷を避けたい
▼ 避けるときフィールド多い / 頻繁書換 / 継承欲しい → Class
─注意: 値型なので関数渡しでコピー発生。ByRef が必要な場面あり
▼ 4. タプル (Tuple)
▼ 目的専用クラス不要の「複数値の組」を一時的に扱う 
Public Function Divide(a As Integer, b As Integer) As (Quotient As Integer, Remainder As Integer)
    Return (a \ b, a Mod b)
End Function

Dim result = Divide(10, 3)
Console.WriteLine(result.Quotient)   ' 3
Console.WriteLine(result.Remainder)  ' 1

' 分解代入
Dim (q, r) = Divide(10, 3)
▼ 利用シーン複数戻り値 / LINQ中間 / 一時集計
─関数から複数値を返したい(ByRefの代替)
─LINQ中間結果のグループ化
─合計と件数のような一時集計
─ 原則「内部・短命」が原則。API公開境界やドメイン中核概念にはクラスを定義すべき
▼ 5. ラムダ式 (Lambda)
▼ 目的関数を「その場で値として」書く 
' 単行ラムダ
Dim square = Function(x As Integer) x * x

' 複数行ラムダ
Dim greet = Function(name As String)
                Dim msg = $"Hello, {name}"
                Return msg.ToUpper()
            End Function

' LINQ で頻出
Dim adults = people.Where(Function(p) p.Age >= 20).
                    Select(Function(p) p.Name).
                    ToList()

' Sub ラムダ(戻り値なし)
Dim log = Sub(s As String) Console.WriteLine($"[LOG] {s}")
▼ 利用シーンLINQ / イベント / 戦略パターン / 非同期
─LINQ(Where, Select, OrderBy)
─イベントハンドラの即席登録
─処理の差し替え(並べ替え基準、フィルタ条件)
─非同期処理の継続(Task.ContinueWith)
─ 必要性ループ+一時変数の処理を、宣言的な「何をしたいか」だけで表現できる(JS arr.map(x => ...) と思想が近い)
▼ 6. 共有メンバー (Shared Member)
▼ 目的クラスそのものに属するメンバー(C#/Java の static) 
Public Class MathHelper
    Public Shared ReadOnly PI As Double = 3.14159

    Public Shared Function Square(x As Double) As Double
        Return x * x
    End Function
End Class

' インスタンス不要で呼べる
Dim r = MathHelper.Square(5)
▼ 利用シーンユーティリティ / 定数 / Factory / Singleton
─ユーティリティ関数(文字列変換、計算ヘルパー)
─定数・設定値
─ファクトリメソッド、シングルトン
─「クラス全体で1つ」の状態(カウンタ等)
▼ 注意点マルチスレッド / テスト容易性
─全インスタンス・全スレッドで共有。マルチスレッドでは排他制御必要
─使い過ぎるとモック差し替え不可。DIで渡せるならインスタンスメソッドが望ましい
▼ 7. Nullable 型 【中核トピック】
▼ なぜ必要か「未入力」と「ゼロ」を分離するため
▼ VB.NET の型は2系統に分かれる
─参照型(Class, String) → Nothing 代入可(参照が空)
─値型(Integer, Date) → Nothing 代入で型のデフォルト値が入る
▼ 業務上の落とし穴 
Dim age As Integer = Nothing   ' これは 0 になる(エラーにならない)

If age = 0 Then
    ' 「0歳」なのか「未入力」なのか区別できない！
End If
▼ 正体ジェネリック構造体 Nullable(Of T) 
Dim a As Integer? = 10
Dim b As Nullable(Of Integer) = 10   ' 同じ意味

' 内部構造
' Nullable(Of T)  ← Structure
'   ├─ HasValue As Boolean   (値が入っているか)
'   └─ value    As T         (実際の値)

「値 + 値があるかのフラグ」の小さな構造体。参照型ではないのでヒープに配置されず軽量。

▼ 基本操作判定・取り出し・Null合体 
Dim age As Integer? = Nothing

' 値の有無確認
If age.HasValue Then ...           ' 推奨
If age IsNot Nothing Then ...      ' これも可

' 値の取り出し
Dim n = age.Value                  ' Nothingなら例外
Dim n = age.GetValueOrDefault()    ' Nothingなら 0
Dim n = age.GetValueOrDefault(-1)  ' Nothingなら -1
Dim n = If(age, -1)                ' Null合体演算子(短い)

注意: If(a, b) は a が Nothing なら b。三項演算子 If(条件, a, b) と区別。

▼ 演算の挙動ハマりどころ
▼ 算術演算は「伝播」する(SQL の NULL と同じ) 
Dim a As Integer? = 10
Dim b As Integer? = Nothing

Dim c = a + b      ' c は Integer? で Nothing
Dim d = a + 5      ' d は Integer? で 15
▼ 比較は要注意 — a = b が False になる 
Dim a As Integer? = Nothing
Dim b As Integer? = Nothing

If a = b Then ...    ' False になる(SQL的にはNULL=NULLは不定)

' 正しい等価判定
If a.Equals(b) Then ...
If Nullable.Equals(a, b) Then ...

意図せず分岐から漏れるバグの温床。Nullable同士の比較は Equals を使う癖を。

▼ Boolean? は3値論理 
Dim x As Boolean? = Nothing
Dim y As Boolean? = True

Dim r1 = x And y   ' Nothing(不定)
Dim r2 = x Or y    ' True (片方が確定で True)

' NG: Nothing は False扱いで分岐される
If someBoolNullable Then ...

' OK: 意図を明示
If someBoolNullable = True Then ...
▼ データ境界DB・フォーム・APIとの変換
▼ DBとのマッピング(DBNull.Value ⇔ Nothing) 
' DB → Nullable
Dim age As Integer? = If(IsDBNull(row("age")), CType(Nothing, Integer?), CInt(row("age")))

' Nullable → DB
cmd.Parameters.AddWithValue("@age", If(age.HasValue, CObj(age.Value), DBNull.Value))

EF Core / Dapper では自動マッピング。生 ADO.NET では DBNull.Value と Nothing は別物。

▼ 文字列パース(空欄を Nothing で受ける) 
Public Function ParseAge(s As String) As Integer?
    Dim n As Integer
    If Integer.TryParse(s, n) Then
        Return n
    Else
        Return Nothing   ' パース失敗を「未入力」として扱う
    End If
End Function
▼ LINQ集計関数で挙動が変わる 
Dim ages As New List(Of Integer?) From {25, Nothing, 30, Nothing, 40}

Dim total = ages.Sum()      ' Nothing は 0 扱いで合計 95
Dim avg = ages.Average()    ' Nothing を除いた平均(31.66...)
Dim cnt = ages.Count()      ' 5(全件)
Dim hasVal = ages.Count(Function(x) x.HasValue)  ' 3

Sum と Average で Nothing の扱いが異なる。Average は分母から除外。

▼ 設計指針使うべき/避けるべきケース
▼ 使うべきとき
─DBのNULL許容列(退職日、完了日、削除日)
─任意入力フォーム項目(電話番号、副次情報)
─未測定・未確定を意味的に区別したい業務データ
─API応答のオプショナルフィールド
▼ 使わない方がいいとき
─「絶対に値が入る」前提の項目
─「0や空文字が未入力」と業務的に決まっている場合(ルール統一する方が一貫)
─パフォーマンス最重要のループ内
▼ 実務パターンDTO / 検索条件 / 集計デフォルト
▼ パターン1: DTO の任意項目 
Public Class EmployeeDto
    Public Property Id As Integer
    Public Property Name As String
    Public Property HiredDate As Date
    Public Property RetiredDate As Date?    ' 在職中は Nothing
    Public Property ManagerId As Integer?   ' 管理職本人は Nothing
End Class
▼ パターン2: 検索条件の任意指定 
Public Function SearchOrders(
    customerId As Integer?,
    fromDate As Date?,
    toDate As Date?
) As List(Of Order)

    Dim query = orders.AsQueryable()
    If customerId.HasValue Then query = query.Where(Function(o) o.CustomerId = customerId.Value)
    If fromDate.HasValue Then query = query.Where(Function(o) o.OrderDate >= fromDate.Value)
    If toDate.HasValue Then query = query.Where(Function(o) o.OrderDate <= toDate.Value)
    Return query.ToList()
End Function

「指定されない条件は無視」が Nullable なら自然に書ける。

▼ パターン3: 集計の安全なデフォルト 
Dim maxScore As Integer = scores.DefaultIfEmpty().Max().GetValueOrDefault(0)

空コレクションで Max() が例外にならないよう保護。

▼ 8. Nullable vs Nothing 比較 【重要】
▼ レイヤーが違う「言葉」と「型」の比較

Nothing → VB.NETのキーワード。文脈で意味が変わる

Nullable → 値型を「Null許容」にするための型(構造体)

比較すると言いつつ、対等な概念ではない。Nullable は Nothing という言葉を借りて「値なし」を表現している。

▼ Nothing の正体文脈で意味が変わる万能キーワード 
Dim s As String = Nothing      ' Null参照(参照がない)
Dim i As Integer = Nothing     ' 0 (Integerのデフォルト値)
Dim b As Boolean = Nothing     ' False
Dim d As Date = Nothing        ' #0001/01/01#
Dim n As Integer? = Nothing    ' Nullable で「値なし」状態
C#VB.NET
参照型に代入nullNothing
値型に代入コンパイルエラーOK(デフォルト値)

VB.NETでは Integer = Nothing が通る。「Nothing = Null」と思い込むと事故る。

▼ 3パターンの違い同じ = Nothing でも結果が違う 
' ケース1: 参照型
Dim s As String = Nothing
If s Is Nothing Then ...        ' 文字列が"未割当"
Console.WriteLine(s.Length)     ' NullReferenceException

' ケース2: 通常の値型
Dim i As Integer = Nothing
Console.WriteLine(i)            ' 0(例外にはならない)
If i = Nothing Then ...         ' True (i = 0 なので)
                                ' 「未入力」のつもりが「0」と区別不能

' ケース3: Nullable
Dim n As Integer? = Nothing
Console.WriteLine(n.HasValue)   ' False
If n Is Nothing Then ...        ' True
Console.WriteLine(n.Value)      ' InvalidOperationException
▼ 「未入力」表現力これが本質的な差 
' 通常の値型 + Nothing → 区別不能
Dim age As Integer = Nothing      ' 0
Dim score As Integer = 0          ' 0 (本物の0点)
' age と score は等しい → 「未入力」と「ゼロ」が混ざる！

' Nullable + Nothing → 区別可能
Dim age As Integer? = Nothing     ' HasValue=False
Dim score As Integer? = 0         ' HasValue=True, Value=0

業務システムでの典型例:

・「血圧 0」と「未測定」
・「在庫 0個」と「未入荷で数量未確定」
・「退職日が今日」と「在職中(退職日なし)」

Nullable は意味を型レベルで分離するための仕組み。

▼ 判定方法型ごとに正しい書き方が違う 
' 参照型のNothing判定
If s Is Nothing Then ...         ' ◎ 推奨
If String.IsNullOrEmpty(s) Then  ' ◎ 空文字も含めて判定

' 値型のNothing判定 → 意味がない
If i Is Nothing Then ...         ' コンパイルエラー
If i = Nothing Then ...          ' i=0 と等価。判定として無意味

' Nullableの判定
If n.HasValue Then ...           ' ◎ 推奨
If n IsNot Nothing Then ...      ' ◎ これも可
▼ アンチパターンやってはいけない書き方 
' NG: 値型のNothingで「未入力」を表現しようとする
Dim age As Integer = Nothing
If age = 0 Then ...    ' 0歳の人と区別できない

' NG: 参照型をわざわざNullableにする
Dim s As Nullable(Of String)   ' そもそもエラー(参照型はNullable不可)

' NG: HasValue確認なしに .Value
Dim v = someNullable.Value     ' Nothing なら例外

' OK
Dim v = someNullable.GetValueOrDefault(0)
Dim v = If(someNullable, 0)
▼ 他言語との対比C# / TypeScript
概念VB.NETC#TypeScript
参照のNullNothingnullnull
値型のNull許容Integer?int?number | null
値型に「Null相当」代入0になるコンパイルエラーエラー(strict)

VB.NETの Nothing は「Null」というより「デフォルト値」に近いと理解すると混乱しない。

▼ 最終まとめ一行で言うと
NothingNullable
正体キーワード(型ではない)構造体(型)
役割「デフォルト値」or「Null参照」を表す万能語値型に「値なし」状態を追加するラッパー
値型での意味デフォルト値(0など)真の「値なし」
「未入力」表現できない(0と混ざる)できる

一行で: Nothing は「言葉」、Nullable は「型」。Nullable は Nothing という言葉を借りて、値型でも意味のある "値なし" を表現できるようにした仕組み。
