00

DOCTYPE・html・head・body

基本

すべてのHTMLページの骨格。<!DOCTYPE html> は標準モード宣言であり、省略すると互換モード（Quirks Mode）でレンダリングされ、CSSの計算が狂う原因になる。

🔰 HTMLドキュメントの全体像
┌──────────────────────────────┐
│ <!DOCTYPE html>              │ ← 標準モード宣言
│ <html lang="ja">             │ ← ルート要素 + 言語指定
│   ┌────────────────────────┐ │
│   │ <head>                 │ │ ← メタ情報（画面に出ない）
│   │   meta, title, link,   │ │
│   │   style, script        │ │
│   └────────────────────────┘ │
│   ┌────────────────────────┐ │
│   │ <body>                 │ │ ← 表示コンテンツ
│   │   header, main, footer │ │
│   └────────────────────────┘ │
│ </html>                      │
└──────────────────────────────┘
HTML — 最小構成テンプレート
<!-- 標準モード宣言：必ず1行目に -->
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>社内システム</title>
</head>
<body>
  <h1>Hello</h1>
</body>
</html>
属性要素説明
lang<html>文書の主言語。ja / en 等。スクリーンリーダーの発音に影響
dir<html>テキスト方向。ltr（左→右）/ rtl
charset<meta>文字エンコーディング。業務系は UTF-8 一択
01

meta タグ徹底解説

基本

<meta> はブラウザ・検索エンジン・SNSに対するメタ情報を宣言する空要素。社内システムでは charset、viewport、CSP、キャッシュ制御が特に重要。

HTML — 業務システム向け meta 一式
<!-- 文字コード（必ず先頭1024バイト以内に） -->
<meta charset="UTF-8">

<!-- レスポンシブ対応 -->
<meta name="viewport"
      content="width=device-width, initial-scale=1.0">

<!-- IE互換モード抑止（社内IE対応レガシー環境） -->
<meta http-equiv="X-UA-Compatible"
      content="IE=edge">

<!-- キャッシュ無効化（開発・テスト時） -->
<meta http-equiv="Cache-Control"
      content="no-cache, no-store, must-revalidate">
<meta http-equiv="Pragma" content="no-cache">

<!-- CSP（Content Security Policy） -->
<meta http-equiv="Content-Security-Policy"
      content="default-src 'self'; script-src 'self'">

<!-- ロボット制御（社内なら通常 noindex） -->
<meta name="robots" content="noindex, nofollow">

<!-- 説明・著者 -->
<meta name="description" content="勤怠管理システム">
<meta name="author" content="IT部門">
name / http-equiv用途業務での重要度
charset文字コード宣言★★★ 必須
viewportモバイル対応★★★ タブレット利用時必須
X-UA-CompatibleIEレンダリング制御★★ レガシー対応
Content-Security-PolicyXSS対策★★★ セキュリティ必須
Cache-Controlキャッシュ制御★★ 開発・運用
robotsクローラ制御★ 社内なら noindex
refresh自動リダイレクト★ ログアウト後のリダイレクト等
02

link・style・script の読み込み

基本

外部リソースの読み込み方法と、パフォーマンスに影響する defer/async の使い分けを理解する。

HTML — リソース読み込みパターン
<!-- 外部CSS -->
<link rel="stylesheet" href="/css/main.css">

<!-- ファビコン -->
<link rel="icon" type="image/svg+xml" href="/favicon.svg">

<!-- プリロード（クリティカルフォント） -->
<link rel="preload" as="font"
      href="/fonts/NotoSansJP.woff2"
      type="font/woff2" crossorigin>

<!-- インラインCSS -->
<style>
  body { margin: 0; }
</style>

<!-- script: 通常 / defer / async -->
<script src="/js/app.js"></script>         <!-- ブロッキング -->
<script src="/js/app.js" defer></script>  <!-- HTML解析後に実行 -->
<script src="/js/analytics.js" async></script> <!-- 非同期DL→即実行 -->
<script type="module" src="/js/esm.js"></script>  <!-- ESモジュール（暗黙defer） -->
🔰 defer vs async の実行タイミング
HTML解析:  ════════════════════▶ DOMContentLoaded
                                       ↓
通常:      ──▶[DL]▶[実行]▶──────────▶  （解析ブロック）
defer:     ──▶[DL並行]──────────▶[実行] （解析後に順序通り実行）
async:     ──▶[DL並行]▶[即実行]──────▶  （DL完了次第、順序不定）
module:    ──▶[DL並行]──────────▶[実行] （defer相当 + import対応）
属性要素説明
rel<link>リソースとの関係（stylesheet / icon / preload / preconnect）
href<link>リソースのURL
as<link rel="preload">リソース種類指定（font / script / style / image）
crossorigin<link> / <script>CORS設定。フォントのpreloadに必要
defer<script>HTML解析完了後に実行。業務系メインJSに推奨
async<script>非同期DL・即実行。アナリティクス系に
type="module"<script>ESモジュール。暗黙的deferかつstrict mode
integrity<script> / <link>SRI（改竄検知ハッシュ）。CDN利用時に推奨
現在: After（推奨）
✓ After: <script src="app.js" defer> — HTML解析と並行してDL。解析完了後に順序通り実行。ユーザには早く画面が表示される。
✗ Before: <script src="app.js">（属性なし）— scriptタグに到達した時点でHTML解析が停止。大きなJSファイルがある場合、白い画面が長く続く。
03

header・nav・main・footer

基本

HTML5セマンティック要素でページ構造を明示する。<main> はページ内に1つだけ。業務システムでは共通ヘッダ・サイドナビ・メインコンテンツ・フッタの4ブロック構成が基本。

HTML — 業務画面の基本レイアウト
<body>
  <header>
    <h1>勤怠管理システム</h1>
    <nav aria-label="メインナビ">
      <ul>
        <li><a href="/dashboard">ダッシュボード</a></li>
        <li><a href="/attendance">打刻</a></li>
        <li><a href="/request">申請</a></li>
      </ul>
    </nav>
  </header>

  <main>
    <!-- ページ固有のコンテンツ -->
    <h2>本日の勤怠</h2>
    <section>...</section>
  </main>

  <footer>
    <p>&copy; 2026 IT部門</p>
  </footer>
</body>
要素暗黙的ARIAロール使用ルール
<header>bannerページ先頭。section/article内では汎用ヘッダ
<nav>navigation主要ナビゲーション。複数ある場合は aria-label で識別
<main>mainページ内に1つだけ。skip-link の飛び先に最適
<footer>contentinfo著作権表示・連絡先等。section内でも使用可
04

section・article・aside

基本

コンテンツの意味的なまとまりを表すセクショニング要素。<section> はテーマを持つ区画、<article> は自己完結コンテンツ、<aside> は補足情報。

HTML — 社内ポータルの構成例
<main>
  <section aria-labelledby="news-heading">
    <h2 id="news-heading">社内ニュース</h2>
    <article>
      <h3>2026年度 健康診断のお知らせ</h3>
      <time datetime="2026-04-15">2026/04/15</time>
      <p>本年度の定期健診は5月から...</p>
    </article>
  </section>

  <aside>
    <h2>クイックリンク</h2>
    <ul>
      <li><a href="/faq">FAQ</a></li>
    </ul>
  </aside>
</main>
要素使いどころ見出し必須？
<section>機能ブロック（検索条件、結果一覧 等）推奨（h2-h6）
<article>お知らせ1件、チケット1件など自己完結推奨
<aside>サイドバー、補足、関連リンク任意
05

h1-h6 と p・br・hr

基本

見出し階層は <h1>→<h6> の順で飛ばさない。<p> は段落、<br> は段落内改行（住所等）、<hr> は話題の区切り。

HTML — 見出し階層の正しい使い方
<h1>経費精算システム</h1>            <!-- ページタイトル: 1つだけ -->
  <h2>新規申請</h2>                    <!-- 機能ブロック -->
    <h3>交通費</h3>                    <!-- サブカテゴリ -->
    <h3>宿泊費</h3>
  <h2>申請履歴</h2>

<!-- ❌ h2 → h4 は飛ばし（アクセシビリティ違反） -->
<h2>承認一覧</h2>
<h4>未承認</h4>  <!-- NG: h3 を飛ばしている -->

<!-- br は住所・詩など「同一段落内の改行」のみ -->
<p>
  〒530-0001<br>
  大阪府大阪市北区梅田1-1-1<br>
  ◯◯ビル 5F
</p>

<!-- hr はトピックの境界線 -->
<hr>
現在: After（推奨）
✓ After（正しい階層）: h1 → h2 → h3 と順序通り。スクリーンリーダーのユーザは見出しジャンプで目的のセクションに素早く移動できる。
✗ Before（見出し飛ばし）: h1 → h3（h2をスキップ）。スクリーンリーダーが「見出しレベル3」と読み上げるが、h2がなく構造が論理的でない。WCAG 2.1 Level A 違反。
06

span・div・インライン装飾タグ

基本

<div> はブロックレベルの汎用コンテナ、<span> はインラインの汎用コンテナ。セマンティックな代替がない場合のみ使う。<strong>、<em>、<mark> 等は意味を持つインライン要素。

HTML — インライン装飾タグの使い分け
<!-- 意味を持つインライン要素 -->
<strong>重要</strong>         <!-- 重要性（太字で表示） -->
<em>強調</em>              <!-- 強調（イタリック） -->
<mark>ハイライト</mark>    <!-- 検索結果のマッチ箇所等 -->
<small>補足</small>       <!-- 注釈・免責 -->
<del>取消線</del>        <!-- 削除されたテキスト -->
<ins>追加テキスト</ins>  <!-- 挿入されたテキスト -->
<code>let x = 1;</code> <!-- インラインコード -->
<kbd>Ctrl+S</kbd>       <!-- キーボード入力 -->
<abbr title="Hyper Text Markup Language">HTML</abbr>
<time datetime="2026-04-15">2026年4月15日</time>

<!-- 汎用コンテナ（最終手段） -->
<div class="card">...</div>
<span class="badge">NEW</span>
要素意味業務での使いどころ
<strong>重要性エラーメッセージ、必須表示
<em>強調注意書きの強調部分
<mark>ハイライト検索結果マッチ、変更差分
<code>コードAPIキー表示、コード例
<time>日時投稿日、期限日、タイムスタンプ
<abbr>略語社内用語の正式名称表示
<del>/<ins>変更履歴改訂履歴、価格変更
07

a タグとリンク属性

中級

<a> はハイパーリンクの要。href に加え、target、rel、download の組み合わせで多彩なリンク動作を制御する。

HTML — a タグの各種パターン
<!-- 基本 -->
<a href="/dashboard">ダッシュボード</a>

<!-- 別タブで開く（外部リンク必須セット） -->
<a href="https://example.com"
   target="_blank"
   rel="noopener noreferrer">外部サイト</a>

<!-- ファイルダウンロード -->
<a href="/reports/q1.pdf"
   download="Q1_report.pdf">📥 Q1レポート</a>

<!-- メールリンク -->
<a href="mailto:support@example.co.jp?subject=問合せ">お問合せ</a>

<!-- 電話（モバイル対応画面） -->
<a href="tel:+81-6-1234-5678">06-1234-5678</a>

<!-- ページ内リンク -->
<a href="#section-results">結果へジャンプ</a>

<!-- JavaScript無効化リンク（SPA内） -->
<a href="#" role="button"
   onclick="handleClick(event)">アクション</a>
属性値の例用途
hrefURL / # / mailto: / tel:リンク先。必須属性
target_blank / _self / _parent開く場所。_blank は必ず rel と併用
relnoopener noreferrerセキュリティ。_blank 時に必須
downloadファイル名クリック時にダウンロード（同一オリジン）
hreflangen / jaリンク先の言語
aria-currentpage / step現在のページを示す（ナビゲーション）
現在: After（推奨）
✓ After: <a href="https://..." target="_blank" rel="noopener noreferrer"> — noopenerにより、開いた新タブから window.opener（元ページ）へのアクセスが遮断される。タブナビング攻撃を防止。
✗ Before: <a href="https://..." target="_blank">（rel省略）— 古いブラウザでは window.opener が参照可能。悪意あるページが元ページのURLを書き換えてフィッシングページに誘導できる。
08

ul・ol・li・dl 定義リスト

基本

リスト要素はナビゲーション、手順、用語集など多用途。<dl>（定義リスト）は詳細画面のラベル+値ペアに最適。

HTML — リスト要素の使い分け
<!-- 順序なしリスト（ナビ、機能一覧） -->
<ul>
  <li>勤怠管理</li>
  <li>経費精算</li>
</ul>

<!-- 順序付きリスト（手順、ランキング） -->
<ol start="1" type="1">
  <li>申請書を作成</li>
  <li value="5">承認者に提出</li>  <!-- 番号を 5 にリセット -->
</ol>

<!-- 定義リスト（詳細画面のラベル+値） -->
<dl>
  <dt>社員番号</dt>
  <dd>EMP-2024-0012</dd>
  <dt>部署</dt>
  <dd>情報システム部</dd>
  <dt>役職</dt>
  <dd>主任</dd>
</dl>
属性要素説明
type<ol>番号種類: 1, a, A, i, I
start<ol>開始番号
reversed<ol>降順（カウントダウン表示）
value<li>ol内で番号をリセット
現在: After（推奨）
✓ After: <dl><dt>社員番号</dt><dd>EMP-001</dd></dl> — ラベルと値のペアを意味的に表現。CSSで横並びレイアウトも可能。tableより軽量で詳細画面に最適。
✗ Before: <p>社員番号: EMP-001</p>（dlなし）— プレーンテキストでは「ラベル」と「値」の関係が機械的に解釈できない。スクリーンリーダーも「用語/定義」として認識しない。

table 構造と属性

中級

業務システムの一覧画面で必須。<table> の構造（thead/tbody/tfoot）とセルの結合（colspan/rowspan）、アクセシビリティ属性（scope/caption）を正しく使う。

HTML — アクセシブルな業務テーブル
<table>
  <caption>2026年4月 勤怠一覧</caption>

  <thead>
    <tr>
      <th scope="col">日付</th>
      <th scope="col">出勤</th>
      <th scope="col">退勤</th>
      <th scope="col">備考</th>
    </tr>
  </thead>

  <tbody>
    <tr>
      <th scope="row">4/1(火)</th>
      <td>09:00</td>
      <td>18:00</td>
      <td>—</td>
    </tr>
    <tr>
      <th scope="row">4/2(水)</th>
      <td colspan="2">有給休暇</td>
      <td>事前申請済</td>
    </tr>
  </tbody>

  <tfoot>
    <tr>
      <th scope="row">合計</th>
      <td colspan="3">出勤: 20日 / 有給: 1日</td>
    </tr>
  </tfoot>
</table>
属性要素説明
scope<th>col / row / colgroup / rowgroup。スクリーンリーダーが見出しを認識
colspan<td> / <th>横方向のセル結合
rowspan<td> / <th>縦方向のセル結合
headers<td>複雑な表で対応するth のidを列挙
現在: After（推奨）
✓ After: caption + scope で構造化されたテーブル

勤怠一覧（2026年4月）
日付 出勤 退勤 備考
4/1(火)09:0018:00—
4/2(水)有給休暇事前申請済
合計出勤: 20日 / 有給: 1日
→ caption で表タイトルが表示され、scope でスクリーンリーダーが列/行の見出しを認識
✗ Before: caption・scope なしの非アクセシブルなテーブル

日付 出勤 退勤 備考
4/1(火)09:0018:00—
4/2(水)有給休暇事前申請済
→ 表のタイトルがなく、日付セルは td なのでスクリーンリーダーが見出しと認識できない

colgroup・col とテーブル装飾

中級

<colgroup>/<col> で列単位のスタイル指定が可能。大量データの一覧表で列幅やハイライトを効率的に制御する。

HTML — colgroup による列スタイル制御
<table>
  <colgroup>
    <col style="width:80px;">          <!-- No列 -->
    <col style="width:200px;">         <!-- 社員名列 -->
    <col span="2"
         style="background:#f0f8ff;">  <!-- 出退勤 2列をまとめてハイライト -->
    <col>                                <!-- 備考列（デフォルト） -->
  </colgroup>
  <thead>
    <tr>
      <th>No</th><th>社員名</th><th>出勤</th><th>退勤</th><th>備考</th>
    </tr>
  </thead>
  <tbody>...</tbody>
</table>
属性要素説明
span<col> / <colgroup>連続する列数をまとめて指定
style / class<col>列単位の幅・背景色を一括制御
現在: After（推奨）
✓ After: colgroup で列幅・背景色を一括制御

No 社員名 出勤 退勤 備考
1山田 太郎09:0018:00—
→ 出勤・退勤列が水色にハイライト。colgroup 1箇所の変更で全行に反映
✗ Before: 各セルに個別で style を付けた冗長な記述

No 社員名 出勤 退勤 備考
1山田 太郎09:0018:00—
→ 全セルに同じ style を繰り返し記述。行が増えるたびに漏れが発生しやすい
11

form タグと method/action

中級

<form> はユーザ入力をサーバへ送信する仕組み。method（GET/POST）、action（送信先）、enctype（エンコーディング）が三大属性。

HTML — form の各パターン
<!-- 基本: POST送信 -->
<form method="POST" action="/api/attendance">
  ...
</form>

<!-- ファイルアップロード（multipart必須） -->
<form method="POST"
      action="/api/upload"
      enctype="multipart/form-data">
  <input type="file" name="document">
  <button type="submit">アップロード</button>
</form>

<!-- 検索フォーム（GET送信 → URLパラメータ） -->
<form method="GET" action="/search" role="search">
  <input type="search" name="q" placeholder="社員名で検索">
  <button type="submit">検索</button>
</form>

<!-- CSRF対策: hidden で token を含める -->
<form method="POST" action="/api/approve">
  <input type="hidden" name="_csrf" value="abc123">
  <button type="submit">承認</button>
</form>
属性値説明
methodGET / POSTGET: URL末尾にパラメータ / POST: リクエストボディ
actionURL送信先。省略するとカレントURL
enctypeapplication/x-www-form-urlencoded（既定）
multipart/form-data

template・slot・dialog

応用

<template> はレンダリングされない雛形HTML。JSで cloneNode してDOMに挿入する。<dialog> はネイティブモーダル/ノンモーダルダイアログ。

HTML + JS — template と dialog
<!-- template: 行テンプレート -->
<template id="row-tmpl">
  <tr>
    <td class="name"></td>
    <td class="dept"></td>
    <td><button type="button">編集</button></td>
  </tr>
</template>

// JS: テンプレートから行を生成
const tmpl = document.getElementById('row-tmpl');
const clone = tmpl.content.cloneNode(true);
clone.querySelector('.name').textContent = '田中太郎';
clone.querySelector('.dept').textContent = 'IT部門';
document.querySelector('tbody').appendChild(clone);

<!-- dialog: ネイティブモーダル -->
<dialog id="confirm-dialog">
  <h2>確認</h2>
  <p>この申請を承認しますか？</p>
  <form method="dialog">
    <button value="cancel">キャンセル</button>
    <button value="confirm">承認</button>
  </form>
</dialog>

// JS: モーダル表示
const dlg = document.getElementById('confirm-dialog');
dlg.showModal();   // モーダル（背面操作不可）
dlg.show();        // ノンモーダル
dlg.close();       // 閉じる
dlg.returnValue;   // "cancel" or "confirm"
✅ dialog のメリット: ::backdrop 疑似要素、ESCキーで閉じる、フォーカストラップがネイティブで実装済み。自前モーダル実装よりアクセシビリティが良い。method="dialog" の form 内 button で returnValue に値がセットされる。

details・summary 折り畳み

基本

<details>+<summary> はネイティブのアコーディオンUI。JS不要で開閉が動く。FAQ、エラー詳細、設定パネルに最適。

HTML — details/summary の各パターン
<!-- 基本: クリックで開閉 -->
<details>
  <summary>エラー詳細を表示</summary>
  <pre>Error: TIMEOUT at /api/attendance (504)
Stack trace: ...</pre>
</details>

<!-- open: 初期状態で開いておく -->
<details open>
  <summary>検索条件</summary>
  <!-- フィルタフォーム -->
</details>

<!-- name 属性: 排他的アコーディオン（同名グループ内で1つだけ開く） -->
<details name="faq">
  <summary>Q1. パスワードを忘れた場合は？</summary>
  <p>情報システム部にご連絡ください。</p>
</details>
<details name="faq">
  <summary>Q2. 打刻を修正するには？</summary>
  <p>勤怠管理 → 修正申請から...</p>
</details>
属性要素説明
open<details>初期状態で開いた状態にする
name<details>同名グループ内で排他的に1つだけ開く（アコーディオン動作）
📘 イベント: details 要素は toggle イベントを発火する。e.target.open で開閉状態を取得できるため、アナリティクスやUI連動に活用可能。
