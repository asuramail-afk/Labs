00

Promiseの3状態を目で見る

基本

Promiseは pending（未決）→ fulfilled（成功）または rejected（失敗）の一方通行。一度決まった状態は二度と変わらない。

🔰 Promiseの状態遷移
Promise は「未来の値を入れる箱」。箱の蓋を開けるまで中身は分からない。 
         new Promise(executor)
                │
           ┌────▼────┐
           │ pending  │  ← 初期状態（まだ結果なし）
           └────┬─────┘
       resolve()│      │reject()
           ┌────▼────┐  ┌────▼────┐
           │fulfilled│  │rejected │
           │ (値あり) │  │(理由あり)│
           └─────────┘  └─────────┘
          ※ どちらか一方にしか遷移しない（不可逆）
promise-states.js
// 1) pending のまま観察
const p1 = new Promise(() => {}); // executor が何もしない
console.log('p1:', p1); // Promise {<pending>}

// 2) fulfilled
const p2 = Promise.resolve(42);
console.log('p2:', p2); // Promise {<fulfilled>: 42}

// 3) rejected
const p3 = Promise.reject('oops');
p3.catch(() => {}); // ← unhandled rejection 防止
console.log('p3:', p3); // Promise {<rejected>: "oops"}

// 4) resolve を2回呼んでも最初の値で確定
const p4 = new Promise(resolve => {
  resolve('first');
  resolve('second'); // 無視される
});
p4.then(v => console.log('p4 value:', v)); // "first"
01

.then() チェーンと値の伝搬

基本

.then() は新しい Promise を返す。コールバックの戻り値が次の .then() の引数になる。Promise を返せばその解決を待ってから次へ進む。

then-chain.js
Promise.resolve(1)
  .then(v => {
    console.log('Step1:', v);   // 1
    return v * 10;              // 普通の値を返す
  })
  .then(v => {
    console.log('Step2:', v);   // 10
    return new Promise(r =>    // Promiseを返す → 解決を待つ
      setTimeout(() => r(v + 5), 500)
    );
  })
  .then(v => {
    console.log('Step3:', v);   // 15 (500ms後)
  })
  .then(v => {
    console.log('Step4:', v);   // undefined (前が何も返さない)
  });
02

.catch() とエラー伝搬

基本

チェーン途中でエラーが発生すると、直近の .catch() まで「飛ぶ」。catch 後のチェーンは正常系として続行する。

catch-propagation.js
Promise.resolve('start')
  .then(v => {
    console.log('A:', v);
    throw new Error('Boom!');
  })
  .then(v => {
    console.log('B: スキップされる');  // ← 実行されない
  })
  .catch(err => {
    console.log('C caught:', err.message); // "Boom!"
    return 'recovered';                  // 正常値を返して復帰
  })
  .then(v => {
    console.log('D:', v);  // "recovered" ← catch後は正常系
  });
パターン挙動使い分け
.then(ok, err) 同じ .then の ok ハンドラのエラーは err で捕捉できない 非推奨（可読性が低い）
.then(ok).catch(err) ok のエラーも catch で捕捉できる 推奨パターン
03

Promise.all vs Promise.allSettled

合成

複数の Promise を並列実行し結果をまとめる2つのAPI。all は1つでも失敗すると即 reject、allSettled は全完了まで待つ。

メソッド1つ失敗したら戻り値の形
Promise.all 即 reject（他は無視） [val1, val2, ...]
Promise.allSettled 全部待ってから結果返却 [{status, value/reason}, ...]
all-vs-allSettled.js
const ok  = ms => new Promise(r => setTimeout(() => r(`OK(${ms}ms)`), ms));
const ng  = ms => new Promise((_, j) => setTimeout(() => j(`NG(${ms}ms)`), ms));

// all: 1つ失敗で即reject
Promise.all([ok(100), ng(200), ok(300)])
  .then(v  => console.log('all OK:', v))
  .catch(e => console.log('all NG:', e));

// allSettled: 全完了まで待つ
Promise.allSettled([ok(100), ng(200), ok(300)])
  .then(results => {
    results.forEach((r, i) =>
      console.log(`  [${i}] ${r.status}:`, r.value ?? r.reason)
    );
  });
04

Promise.race vs Promise.any

合成

race は最初に決着した結果（成功でも失敗でも）を返す。any は最初の成功を返す。

メソッド最初に返すもの全失敗時
Promise.race 最初の settled（成功 or 失敗） 最初の reject
Promise.any 最初の fulfilled（成功のみ） AggregateError
race-vs-any.js
const ok  = ms => new Promise(r => setTimeout(() => r(`OK(${ms}ms)`), ms));
const ng  = ms => new Promise((_, j) => setTimeout(() => j(`NG(${ms}ms)`), ms));

// race: 最速=ng(50ms) → reject
Promise.race([ok(200), ng(50), ok(300)])
  .then(v  => console.log('race OK:', v))
  .catch(e => console.log('race NG:', e));

// any: 失敗をスキップし最初の成功=ok(200ms)
Promise.any([ng(50), ok(200), ng(100)])
  .then(v  => console.log('any OK:', v))
  .catch(e => console.log('any NG:', e.errors));

// any: 全失敗 → AggregateError
Promise.any([ng(50), ng(100)])
  .then(v  => console.log('any2 OK:', v))
  .catch(e => console.log('any2 NG: AggregateError', e.errors));
05

async/await 基本構文

基本

async 関数は常にPromiseを返す。await はPromiseの解決を待ち、値を取り出す構文糖衣。

🔰 then チェーンとの対応
// .then() チェーン          // async/await
fetch(url)                   async function getData() {
  .then(r => r.json())         const r = await fetch(url);
  .then(data => {              const data = await r.json();
    console.log(data);          console.log(data);
  })                          }
  .catch(e => { ... });      // try-catch でエラー処理
async-await-basics.js
async function demo() {
  console.log('1: start');

  const result = await new Promise(r =>
    setTimeout(() => r('hello'), 300)
  );
  console.log('2: got', result);

  // async関数の戻り値は自動的にPromiseでラップ
  return result.toUpperCase();
}

console.log('A: before call');
demo().then(v => console.log('3: final', v));
console.log('B: after call');
// 出力順: A → 1 → B → 2 → 3
06

await の逐次 vs 並列実行

応用

await を直列に並べると逐次実行。Promise.all と組み合わせると並列実行になり、合計時間が短縮される。

sequential-vs-parallel.js
const wait = (ms, label) =>
  new Promise(r => setTimeout(() => r(label), ms));

// ❌ 逐次: 300+300+300 = 約900ms
async function sequential() {
  const t0 = Date.now();
  const a = await wait(300, 'A');
  const b = await wait(300, 'B');
  const c = await wait(300, 'C');
  console.log(`逐次: [${a},${b},${c}] ${Date.now()-t0}ms`);
}

// ✅ 並列: max(300,300,300) = 約300ms
async function parallel() {
  const t0 = Date.now();
  const [a, b, c] = await Promise.all([
    wait(300, 'A'),
    wait(300, 'B'),
    wait(300, 'C'),
  ]);
  console.log(`並列: [${a},${b},${c}] ${Date.now()-t0}ms`);
}

sequential().then(() => parallel());
07

実戦エラーハンドリング

応用

try/catch で await のエラーを捕捉。finally でローディング解除。並列時は allSettled で部分的成功を拾う実戦パターン。

error-handling-patterns.js
// パターン1: 基本 try-catch-finally
async function fetchData() {
  let loading = true;
  console.log('loading:', loading);
  try {
    const data = await Promise.reject(new Error('Network error'));
    console.log('data:', data);
  } catch (e) {
    console.log('caught:', e.message);
  } finally {
    loading = false;
    console.log('loading:', loading, '(cleanup done)');
  }
}

// パターン2: allSettled で部分成功を活かす
async function multiAPI() {
  const results = await Promise.allSettled([
    Promise.resolve({ user: 'Tom' }),
    Promise.reject(new Error('Profile API down')),
    Promise.resolve({ prefs: { theme: 'dark' } }),
  ]);
  const ok = results.filter(r => r.status === 'fulfilled');
  const ng = results.filter(r => r.status === 'rejected');
  console.log(`成功: ${ok.length}, 失敗: ${ng.length}`);
  ok.forEach(r => console.log('  ✅', JSON.stringify(r.value)));
  ng.forEach(r => console.log('  ❌', r.reason.message));
}

fetchData().then(() => multiAPI());
