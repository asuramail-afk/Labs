00

for 文（カウンタ型）

基本

最も基本的なループ。初期化; 条件; 更新 の3式で制御する。 break / continue / ラベル付きジャンプがすべて使える。

🔰 for ループの途中経過（i の変化）
3式は ①初期化 → ②条件 → ③本体 → ④更新 → ②条件 … の順に回る。 
  for (let i = 0; i < 5; i++) { console.log(i); }

  時点     i の値   i<5 判定   本体実行        次の i
  ─────    ──────  ──────────  ───────────────  ─────
  ①初期化  0        —           —                —
  ②反復1   0        true        console.log(0)   1
  ②反復2   1        true        console.log(1)   2
  ②反復3   2        true        console.log(2)   3
  ②反復4   3        true        console.log(3)   4
  ②反復5   4        true        console.log(4)   5
  ②反復6   5        false       — 脱出            —

  ● break: 次の「②条件」に行かず即終了
  ● continue: 「③本体の残り」をスキップして「④更新」へ
JavaScript
// 基本: 0〜4 を出力
for (let i = 0; i < 5; i++) {
  console.log(`i = ${i}`);
}

// 逆順ループ
for (let i = 4; i >= 0; i--) {
  console.log(`逆順: ${i}`);
}

// break でループ脱出
for (let i = 0; i < 10; i++) {
  if (i === 3) break;
  console.log(`break前: ${i}`);
}

// continue で偶数スキップ
for (let i = 0; i < 6; i++) {
  if (i % 2 === 0) continue;
  console.log(`奇数のみ: ${i}`);
}
01

while 文

基本

条件が true の間ループし続ける。条件が最初から偽なら1度も実行されない。 カウンタ更新を忘れると無限ループになるので要注意。

🔰 while の途中経過
条件判定 → 本体 → 条件判定…の繰り返し。本体内でカウンタを進めないと永遠ループ。 
  let count = 0;
  while (count < 5) { console.log(count); count++; }

  判定前 count  count<5   本体               判定後 count
  ────────────  ───────   ────────────────   ────────────
  0             true      log(0), count++   → 1
  1             true      log(1), count++   → 2
  2             true      log(2), count++   → 3
  3             true      log(3), count++   → 4
  4             true      log(4), count++   → 5
  5             false     — 脱出             —

  ● 2つ目の例: let x = 10; while (x < 5) { ... }
       初回判定で false → 本体は1度も実行されず終わる
JavaScript
let count = 0;
while (count < 5) {
  console.log(`count = ${count}`);
  count++;
}

// 条件が最初から偽 → 実行されない
let x = 10;
while (x < 5) {
  console.log('これは出力されない');
  x++;
}
console.log(`while終了後 x = ${x}`);
02

do...while 文

基本

while と異なり、最低1回は必ず実行される。条件判定がループ末尾にある後判定ループ。

🔰 do...while は「実行してから判定」
while は「判定→本体」、do...while は「本体→判定」。初回は条件に関わらず必ず1回動くのが違い。 
  let n = 10;
  do { console.log(n); n++; } while (n < 5);

  フェーズ        n    本体実行          n<5判定
  ────────────    ──   ──────────────    ───────
  初回本体        10   log(10), n++     → (次へ)
  初回判定後     11   —                false → 脱出

  結果: n=10 が 1回だけログ出力される（while だと 0回）

  比較: let m = 0; do { log(m); m++; } while (m < 3);
  本体→判定 繰り返し:
    m=0 log → m=1 true
    m=1 log → m=2 true
    m=2 log → m=3 false → 脱出
JavaScript
// 条件が偽でも1回は実行される
let n = 10;
do {
  console.log(`do...while: n = ${n}`);
  n++;
} while (n < 5);

console.log(`→ 最終 n = ${n} (1回だけ実行された)`);

// 通常の繰返し
let m = 0;
do {
  console.log(`ループ中: m = ${m}`);
  m++;
} while (m < 3);
03

for...in（プロパティ列挙）

中級

オブジェクトの列挙可能なプロパティ名（キー）を反復する。 プロトタイプチェーン上のプロパティも列挙されるため、hasOwnProperty でのガードが推奨。 配列には使わないのがベストプラクティス（インデックスが文字列で返り、順序が保証されない場合がある）。

🔰 for...in は「キー」が1個ずつ key に入る
値ではなくキー(プロパティ名)が取れる。欲しい値は obj[key] で引き出す必要がある。 
  const user = { name:'Tom', age:40, city:'Osaka' };
  for (const key in user) { console.log(key, user[key]); }

  反復   key         user[key]
  ────   ─────────   ─────────
  1      "name"      "Tom"
  2      "age"       40
  3      "city"      "Osaka"

  ⚠ プロトタイプも拾う:
    function Animal(name){ this.name = name; }
    Animal.prototype.legs = 4;
    const dog = new Animal('Pochi');

    反復   key       dog[key]   出所
    ────   ───────   ────────   ────────────
    1      "name"    "Pochi"    own （dog 自身）
    2      "legs"    4          prototype （継承）← 意図外なら hasOwnProperty で弾く

  ⚠ 配列に使うと idx が文字列 ("0","1","2")。算術すると文字連結事故の元。
JavaScript
const user = { name: 'Tom', age: 40, city: 'Osaka' };

// キーを列挙
for (const key in user) {
  console.log(`${key}: ${user[key]}`);
}

// ⚠ 配列に for...in → インデックスが「文字列」
const arr = ['a', 'b', 'c'];
for (const idx in arr) {
  console.log(`idx="${idx}" (型: ${typeof idx})`);
}

// hasOwnProperty でプロトタイプ除外
function Animal(name) { this.name = name; }
Animal.prototype.legs = 4;
const dog = new Animal('Pochi');

for (const key in dog) {
  const own = dog.hasOwnProperty(key) ? 'own' : 'prototype';
  console.log(`${key} = ${dog[key]} [${own}]`);
}

for...of（イテラブル反復）

中級

Symbol.iterator を持つイテラブル（配列、文字列、Map、Set、NodeList 等）の値を反復。 for...in と違い、キーではなく値が返る。break / continue が使える。

🔰 for...of は「値」を直接もらえる
舞台裏では [Symbol.iterator]() を呼び、.next() が返す {value, done} の value だけを fruit に入れる。 
  const fruits = ['🍎','🍌','🍇'];
  for (const fruit of fruits) { console.log(fruit); }

  内部で起きていること（ざっくり）:
    iter = fruits[Symbol.iterator]();
    ┌─────────────────────────────────────┐
    │ step1: iter.next() → {value:'🍎', done:false}  fruit = '🍎'  log   │
    │ step2: iter.next() → {value:'🍌', done:false}  fruit = '🍌'  log   │
    │ step3: iter.next() → {value:'🍇', done:false}  fruit = '🍇'  log   │
    │ step4: iter.next() → {value:undefined, done:true}  脱出              │
    └─────────────────────────────────────┘

  文字列 "大阪🏯" で反復:   "大" → "阪" → "🏯"
     ⚠ length で回すと 🏯 は2単位（サロゲートペア）のため壊れるが、for...of は安全。

  break で途中脱出:
    n=10 (true→log),  n=20 (true→log),  n=30 (>25 で break → 脱出)
JavaScript
// 配列の値を反復
const fruits = ['🍎', '🍌', '🍇'];
for (const fruit of fruits) {
  console.log(fruit);
}

// 文字列の文字を反復（サロゲートペア安全）
for (const ch of '大阪🏯') {
  console.log(ch);
}

// break で途中脱出
for (const n of [10, 20, 30, 40]) {
  if (n > 25) break;
  console.log(`値: ${n}`);
}

// ⚠ 普通のオブジェクトは iterable ではない → エラー
// for (const v of {a:1}) {} // TypeError!
console.log('※ 普通のオブジェクトは for...of 不可（TypeError）');
05

forEach（配列メソッド）

基本

Array.prototype.forEach はコールバック関数を各要素に適用する。 break / continue は使えない。return は現在の反復をスキップするだけ（ループ全体は止まらない）。 戻り値は常に undefined。

🔰 forEach のコールバック引数は3つ
forEach((要素, インデックス, 配列本体) => {...}) — 使うのは多くの場合1つ目だけ。 
  ['red','green','blue'].forEach((color, i, arr) => { ... });

  反復   color      i    arr                        コールバック本体で出力
  ────   ────────   ──   ───────────────────────   ──────────────────────
  1      'red'      0    ['red','green','blue']    "[0] red  (配列長: 3)"
  2      'green'    1    ['red','green','blue']    "[1] green (配列長: 3)"
  3      'blue'     2    ['red','green','blue']    "[2] blue (配列長: 3)"

  ⚠ 戻り値と制御の違い:
    ・ return したら「そのコールバックが終わる」だけ → 次の反復は止まらない（continue 相当）
    ・ break は書けない（SyntaxError） → 途中脱出したいなら for...of / some / every を使う
    ・ forEach 全体の戻り値は 必ず undefined（新配列を作りたいなら map）
JavaScript
const colors = ['red', 'green', 'blue'];

// 基本: (要素, インデックス, 配列本体)
colors.forEach((color, i, arr) => {
  console.log(`[${i}] ${color}  (配列長: ${arr.length})`);
});

// return は continue 相当（ループは止まらない）
[1,2,3,4,5].forEach(n => {
  if (n === 3) return; // 3 だけスキップ
  console.log(`n = ${n}`);
});

// ⚠ break は使えない（構文エラー）
console.log('※ forEach 内で break は SyntaxError');

// 戻り値は undefined
const result = [1,2].forEach(x => x * 2);
console.log(`forEach戻り値: ${result}`);

map（変換して新配列を返す）

基本

各要素にコールバックを適用し、戻り値で新しい配列を生成する。 元の配列は変更しない（非破壊）。forEach との最大の違いは「戻り値がある」こと。

🔰 map は「要素を変換して新しい配列を作る」
各反復の コールバック戻り値が、同じ位置に詰め込まれた新配列を返す。元配列は無傷。 
  const nums = [1,2,3,4];
  const doubled = nums.map(n => n * 2);

  反復    n     戻り値(n*2)     doubled（作成途中）
  ────    ──    ──────────     ──────────────────
  1       1     2               [2]
  2       2     4               [2, 4]
  3       3     6               [2, 4, 6]
  4       4     8               [2, 4, 6, 8]  ← 完成

  nums ──────────── そのまま [1,2,3,4]（変更なし・非破壊）
  doubled ───────── [2,4,6,8]（新規メモリに作られた別配列）

  オブジェクト配列の射影も同じ: users.map(u => u.name)
    反復1: u={name:'Alice',score:80}  → 'Alice'
    反復2: u={name:'Bob',score:95}    → 'Bob'
    → names = ['Alice','Bob']
JavaScript
const nums = [1, 2, 3, 4];

// 2倍にした新配列
const doubled = nums.map(n => n * 2);
console.log('元配列:', nums);
console.log('map結果:', doubled);

// オブジェクト配列の変換
const users = [
  { name: 'Alice', score: 80 },
  { name: 'Bob',   score: 95 },
];
const names = users.map(u => u.name);
console.log('名前だけ抽出:', names);

// インデックスも使える
const indexed = ['a','b','c'].map((v, i) => `${i}:${v}`);
console.log(indexed);

filter（条件で抽出）

基本

コールバックが true を返す要素だけを集めた新しい配列を返す。 元の配列は変更されない。条件に合う要素がなければ空配列 []。

🔰 filter は「通した人だけ入るふるい」
コールバック戻り値が truthy なら残す、falsy なら捨てる。要素の中身自体は変換しない（そこがmapとの違い）。 
  const scores = [45, 82, 73, 91, 58, 66];
  const passed = scores.filter(s => s >= 70);

  反復   s     s>=70     採用?     passed（作成途中）
  ────   ──    ───────   ───────   ──────────────────
  1      45    false     捨てる    []
  2      82    true      残す      [82]
  3      73    true      残す      [82, 73]
  4      91    true      残す      [82, 73, 91]
  5      58    false     捨てる    [82, 73, 91]
  6      66    false     捨てる    [82, 73, 91]  ← 完成

  scores ──── 元のまま [45,82,73,91,58,66]
  passed ──── [82, 73, 91]

  空配列になるケース: filter(s => s === 100) → []

  テクニック: .filter(Boolean)
    falsy（0, "", null, undefined, false, NaN）を一括除去。
    [0,"",null,"hello",undefined,42,false] → ["hello", 42]
JavaScript
const scores = [45, 82, 73, 91, 58, 66];

// 70以上を抽出
const passed = scores.filter(s => s >= 70);
console.log('合格者:', passed);

// 条件不一致 → 空配列
const perfect = scores.filter(s => s === 100);
console.log('満点者:', perfect);

// falsy 除去テクニック
const mixed = [0, '', null, 'hello', undefined, 42, false];
const truthy = mixed.filter(Boolean);
console.log('truthy のみ:', truthy);

reduce（集約・畳み込み）

中級

配列を単一の値に集約する。アキュムレータ（累積値）と現在の要素を受け取り、最終結果を返す。 初期値を省略すると先頭要素が初期値になるが、空配列で TypeError が出るため常に初期値を渡すのが安全。

🔰 reduce のデータフロー（途中経過の蓄積）
acc（累積値）はバケツ。各反復で (acc, cur) → 次の acc を返す。最終的なバケツの中身が戻り値。 
  [10, 20, 30].reduce((acc, cur) => acc + cur, 0);

  初期値 acc = 0
  反復   acc(入力)   cur    コールバック戻り値   次の acc
  ────   ─────────   ────   ──────────────────   ────────
  1      0           10     0 + 10 = 10          10
  2      10          20     10 + 20 = 30         30
  3      30          30     30 + 30 = 60         60 ← reduce の戻り値

  オブジェクトをバケツにした集計（グルーピング）:
    items = ['apple','banana','apple','cherry','banana','apple']
    初期値 = {}

    反復   item       acc（更新前）              acc（更新後）
    ────   ────────   ──────────────────────    ──────────────────────
    1      'apple'    {}                        {apple:1}
    2      'banana'   {apple:1}                 {apple:1, banana:1}
    3      'apple'    {apple:1, banana:1}       {apple:2, banana:1}
    4      'cherry'   {apple:2, banana:1}       {apple:2, banana:1, cherry:1}
    5      'banana'   {apple:2, banana:1, cherry:1}  {apple:2, banana:2, cherry:1}
    6      'apple'    {apple:2, banana:2, cherry:1}  {apple:3, banana:2, cherry:1}

  ⚠ 初期値を省略すると、先頭要素が初期 acc になる → 空配列で TypeError。必ず第2引数を。
JavaScript
// 合計
const sum = [10, 20, 30].reduce((acc, cur) => {
  console.log(`  acc=${acc}, cur=${cur}`);
  return acc + cur;
}, 0);
console.log(`合計: ${sum}`);

// オブジェクトへの集約（グルーピング）
const items = ['apple','banana','apple','cherry','banana','apple'];
const counts = items.reduce((obj, item) => {
  obj[item] = (obj[item] || 0) + 1;
  return obj;
}, {});
console.log('出現回数:', JSON.stringify(counts));

// 最大値
const max = [3,7,1,9,4].reduce((a, b) => a > b ? a : b);
console.log(`最大値: ${max}`);

some / every（存在・全称判定）

基本

some: 1つでも条件を満たせば true（短絡評価で即停止）。 every: 全要素が条件を満たせば true（1つでも偽で即停止）。

🔰 some / every は「見つけ次第/見破り次第」止まる
短絡評価。最後まで回さずに結論が出たら止まる → 重い判定でも効率が良い。 
  const ages = [15, 22, 17, 30];

  ● some: 1つでも成人(>=18) あるか?
    反復   a    a>=18     動作
    ────   ──   ───────   ──────────────────
    1      15   false     続行
    2      22   true      即停止 → some は true
    (17, 30 は評価されない)

  ● every: 全員 成人?
    反復   a    a>=18     動作
    ────   ──   ───────   ──────────────────
    1      15   false     即停止 → every は false
    (22, 17, 30 は評価されない)

  ● 空配列の特殊ルール
    [].some(…)  → 常に false（1人もいないので「1人でも」は成立しない）
    [].every(…) → 常に true （1人もいないので反証できない＝空真）
JavaScript
const ages = [15, 22, 17, 30];

// some: 1人でも成人がいるか？
const hasAdult = ages.some(a => {
  console.log(`  some チェック: ${a}`);
  return a >= 18;
});
console.log(`成人がいる: ${hasAdult}`);

// every: 全員が成人か？
const allAdult = ages.every(a => {
  console.log(`  every チェック: ${a}`);
  return a >= 18;
});
console.log(`全員成人: ${allAdult}`);

// 空配列の挙動
console.log(`空.some:  ${[].some(() => true)}`);  // false
console.log(`空.every: ${[].every(() => false)}`); // true（空真）

find / findIndex（最初の一致を検索）

基本

find: 条件を満たす最初の要素を返す（なければ undefined）。 findIndex: 条件を満たす最初のインデックスを返す（なければ -1）。

🔰 find は「最初のヒットで即返す」
filter が「全部のふるい」なのに対し、find は「最初の1個だけ」。見つけたら残りはスキップ。 
  products = [
    {id:1, name:'ペン',   price:100},
    {id:2, name:'ノート', price:250},
    {id:3, name:'消しゴム', price:80},
  ];

  products.find(p => p.price >= 200);

  反復   p                            p.price>=200   動作
  ────   ───────────────────────────  ─────────────  ───────────────────────
  1      {id:1, ペン, 100}            false          続行
  2      {id:2, ノート, 250}          true           即停止 → このオブジェクトを返す
  (id:3 は評価されない)

  戻り値: {id:2, name:'ノート', price:250}

  findIndex:
    戻り値 = 2 （配列のインデックス）

  見つからない時:
    find      → undefined
    findIndex → -1

  ES2023 findLast / findLastIndex:
    nums = [1,2,3,4,5]
    後ろから探索: 5 (奇)→3 (奇)→... (省略) ... 4 (偶)でヒット → 4 を返す
JavaScript
const products = [
  { id: 1, name: 'ペン',    price: 100 },
  { id: 2, name: 'ノート',  price: 250 },
  { id: 3, name: '消しゴム', price: 80  },
];

// find: 最初に200円以上の商品
const found = products.find(p => p.price >= 200);
console.log('find:', found);

// findIndex: インデックスを取得
const idx = products.findIndex(p => p.name === '消しゴム');
console.log(`findIndex: ${idx}`);

// 見つからない場合
const notFound = products.find(p => p.price > 1000);
console.log(`見つからない: ${notFound}`); // undefined

// findLast / findLastIndex（ES2023）
const nums = [1,2,3,4,5];
const lastEven = nums.findLast(n => n % 2 === 0);
console.log(`findLast (偶数): ${lastEven}`);
11

flatMap（展開＋変換）

中級

map + flat(1) を1ステップで行う。 各要素から配列を返すと結果を1段階フラットにする。要素の除去（空配列を返す）にも使える。

🔰 flatMap = map の戻り値(配列) を 1 段階ほどく
map は「配列の中に配列が入ったまま」、flatMap は「1段だけ平らにならす」。 
  ['hello world', 'foo bar baz'].flatMap(s => s.split(' '));

  反復   s              split(' ') の戻り値         展開後の累積
  ────   ─────────────  ──────────────────────────  ───────────────────────────────
  1      'hello world'  ['hello', 'world']           ['hello', 'world']
  2      'foo bar baz'  ['foo', 'bar', 'baz']        ['hello', 'world', 'foo', 'bar', 'baz']

  map だと: [['hello','world'], ['foo','bar','baz']]  ← 入れ子のまま
  flatMap:  ['hello','world','foo','bar','baz']      ← 1段平ら

  ● 空配列を返すと「除去」できる（filter + map の合体技）
    [1,2,3,4,5].flatMap(n => n%2===0 ? [n] : []);

    反復   n    戻り値     累積
    ────   ──   ────────   ─────────
    1      1    []         []
    2      2    [2]        [2]
    3      3    []         [2]
    4      4    [4]        [2, 4]
    5      5    []         [2, 4]  ← 偶数だけ残った
JavaScript
// 文を単語に分割して1次元化
const sentences = ['hello world', 'foo bar baz'];
const words = sentences.flatMap(s => s.split(' '));
console.log(words);

// 要素の除去（偶数だけ残す）
const nums = [1,2,3,4,5];
const evens = nums.flatMap(n => n % 2 === 0 ? [n] : []);
console.log('偶数のみ:', evens);

// 1対多の展開
const data = [{ tag: 'a', vals: [1,2] }, { tag: 'b', vals: [3] }];
const flat = data.flatMap(d => d.vals.map(v => ({ tag: d.tag, val: v })));
console.log(JSON.stringify(flat));

entries / keys / values

中級

配列・オブジェクトの entries(), keys(), values() で イテレータを取得し、for...of と組み合わせる。 オブジェクトには Object.entries() 等の静的メソッドを使う。

🔰 entries / keys / values が返すペアと分割代入
entries() は [key, value] の組を返すイテレータ。分割代入 [i, v] で両方同時に受け取れる。 
  配列の arr.entries()
    arr = ['🍎','🍌','🍇']

    反復   iter.next().value   分割代入後      出力
    ────   ─────────────────   ─────────────   ───────────
    1      [0, '🍎']            i=0, v='🍎'    "[0] 🍎"
    2      [1, '🍌']            i=1, v='🍌'    "[1] 🍌"
    3      [2, '🍇']            i=2, v='🍇'    "[2] 🍇"

  オブジェクトは Object.entries(obj) で [key, value] 配列に変換
    config = {host:'localhost', port:3000, debug:true}

    反復   [key, val]                   出力
    ────   ──────────────────────────   ──────────────────
    1      ['host', 'localhost']        "host → localhost"
    2      ['port', 3000]               "port → 3000"
    3      ['debug', true]              "debug → true"

  keys()     → ['host','port','debug']
  values()   → ['localhost', 3000, true]
  entries()  → [['host','localhost'], ['port',3000], ['debug',true]]  ← 両方のセット
JavaScript
// 配列の entries()
const arr = ['🍎', '🍌', '🍇'];
for (const [i, v] of arr.entries()) {
  console.log(`[${i}] ${v}`);
}

// オブジェクトの Object.entries()
const config = { host: 'localhost', port: 3000, debug: true };
for (const [key, val] of Object.entries(config)) {
  console.log(`${key} → ${val}`);
}

// Object.keys / Object.values
console.log('keys:', Object.keys(config));
console.log('values:', Object.values(config));

Map / Set の反復

中級

Map と Set はイテラブルなので for...of と forEach の両方が使える。 Map の for...of では [key, value] の分割代入が便利。

🔰 Map / Set は「挿入順」を保つコレクション
普通のオブジェクトと違い、追加した順番どおりに反復される。キーが文字列に強制変換もされない。 
  const map = new Map([
    ['name', 'Tom'],
    ['lang', 'JavaScript'],
    ['city', 'Osaka'],
  ]);

  for (const [k, v] of map) { ... }

  反復   iter.next().value            [k, v] 分割代入        出力
  ────   ──────────────────────────   ──────────────────    ─────────────────────
  1      ['name', 'Tom']              k='name', v='Tom'     "Map: name = Tom"
  2      ['lang', 'JavaScript']       k='lang', v='JS'      "Map: lang = JavaScript"
  3      ['city', 'Osaka']            k='city', v='Osaka'   "Map: city = Osaka"

  Map.forEach は (value, key) の順で来る ← Map でも 配列と揃えるためこの順
    forEach((val, key) => ...) → "name → Tom"

  ● Set は重複を自動排除
    new Set([1, 2, 3, 2, 1])
      内部的に  →  {1, 2, 3}   （重複 2, 1 は黙って捨てられる）
      size = 3
    反復: 1 → 2 → 3
JavaScript
// Map の反復
const map = new Map([
  ['name', 'Tom'],
  ['lang', 'JavaScript'],
  ['city', 'Osaka'],
]);

for (const [k, v] of map) {
  console.log(`Map: ${k} = ${v}`);
}

map.forEach((val, key) => {
  console.log(`Map.forEach: ${key} → ${val}`);
});

// Set の反復
const set = new Set([1, 2, 3, 2, 1]);
console.log(`Set サイズ: ${set.size}`);
for (const v of set) {
  console.log(`Set: ${v}`);
}

ジェネレータと for...of

上級

function* で定義するジェネレータは yield で値を逐次返すイテラブル。 for...of で消費でき、無限シーケンスも安全に扱える（消費側で break する）。

🔰 ジェネレータ＝「yield で一時停止する関数」
普通の関数は最後まで一気に走る。ジェネレータは yield で一旦止まり、次の .next() まで待機する。 
  function* range(start, end) {
    for (let i = start; i < end; i++) { yield i; }
  }

  const it = range(3, 7);   ← この時点ではまだ何も動かない（生成器オブジェクトを返すだけ）

  イベント            ジェネレータ内の状態   .next() の戻り値
  ─────────────────   ─────────────────────  ──────────────────
  it.next()            i=3 で yield 停止       {value:3, done:false}
  it.next()            i=4 で yield 停止       {value:4, done:false}
  it.next()            i=5 で yield 停止       {value:5, done:false}
  it.next()            i=6 で yield 停止       {value:6, done:false}
  it.next()            ループ終了・関数終了    {value:undefined, done:true}

  for (const n of range(3,7)) は上を自動で回して n = 3,4,5,6 をくれる。

  ● 無限ジェネレータは「消費側が止める」
    function* fibonacci() { let a=0,b=1; while(true) { yield a; [a,b]=[b,a+b]; } }

    発行回数  yield した値 (a)      更新後 [a, b]
    ────────  ────────────────      ─────────────
    1         0                     [1, 1]
    2         1                     [1, 2]
    3         1                     [2, 3]
    4         2                     [3, 5]
    5         3                     [5, 8]
    …（消費側で break するまで止まらない）

  ⚠ [...fibonacci()] とすると無限にメモリを食うので禁止。有限の range(0,5) だけ OK。
JavaScript
// 有限ジェネレータ
function* range(start, end) {
  for (let i = start; i < end; i++) {
    yield i;
  }
}

for (const n of range(3, 7)) {
  console.log(`range: ${n}`);
}

// 無限ジェネレータ（break で安全に停止）
function* fibonacci() {
  let a = 0, b = 1;
  while (true) {
    yield a;
    [a, b] = [b, a + b];
  }
}

let count = 0;
for (const fib of fibonacci()) {
  if (count++ >= 10) break; // 安全に停止
  console.log(`fib[${count-1}] = ${fib}`);
}

// スプレッドで配列化（有限のみ！）
const arr = [...range(0, 5)];
console.log('スプレッド:', arr);

for await...of（非同期イテレータ）

上級

async function* で作る非同期ジェネレータや、Promiseの配列を for await...of で順次消費する。API のページング処理等で活躍。

🔰 for await...of = 「await を挟みながら回る for...of」
通常の for...of は即値を処理するが、for await は各反復で Promise を await して値を取り出す。 
  async function* asyncRange(start, end) {
    for (let i = start; i < end; i++) {
      await new Promise(r => setTimeout(r, 100));  ← 100ms 待つ
      yield i;
    }
  }

  for await (const n of asyncRange(0, 4)) { log(n); }

  経過時間   ジェネレータ状態    for await 受け取る値    出力
  ────────   ──────────────────  ───────────────────    ──────────
  0ms        awaiting…            —                       —
  100ms      yield 0              n=0                      "async: 0"
  200ms      yield 1              n=1                      "async: 1"
  300ms      yield 2              n=2                      "async: 2"
  400ms      yield 3              n=3                      "async: 3"
  400ms      done:true            —                        脱出

  ● Promise 配列を for await...of で逐次消費
    [Promise.resolve('A'), Promise.resolve('B'), Promise.resolve('C')]

    反復   await 前           await 後（解決値）
    ────   ───────────────    ──────────────────
    1      Promise<'A'>      val = 'A'
    2      Promise<'B'>      val = 'B'
    3      Promise<'C'>      val = 'C'

    ⚠ 並列ではない: 前の Promise が解決してから次を await する（順次処理）。
    並列にしたいなら Promise.all(arr) を先に通す。
JavaScript
// 非同期ジェネレータ
async function* asyncRange(start, end) {
  for (let i = start; i < end; i++) {
    await new Promise(r => setTimeout(r, 100));
    yield i;
  }
}

(async () => {
  for await (const n of asyncRange(0, 4)) {
    console.log(`async: ${n}`);
  }

  // Promise配列を for await...of で逐次処理
  const promises = [
    Promise.resolve('A'),
    Promise.resolve('B'),
    Promise.resolve('C'),
  ];
  for await (const val of promises) {
    console.log(`Promise値: ${val}`);
  }
  console.log('非同期処理完了');
})();

再帰によるループ代替

上級

ループの代わりに関数が自分自身を呼ぶパターン。ツリー走査やネスト構造の処理で必須。 終了条件（ベースケース）を必ず定義しないとスタックオーバーフローになる。

🔰 再帰は「自分を呼ぶ → 終了条件で戻ってきて掛け算」
呼び出し時に引数を積み上げ、ベースケースで折り返して戻り値を掛け合わせて帰ってくる（コールスタック）。 
  factorial(5) の流れ:

  【行きの呼び出し（スタックに積む）】
     factorial(5)  →  5 * factorial(4)         ← 結果待ち
        factorial(4)  →  4 * factorial(3)      ← 結果待ち
           factorial(3)  →  3 * factorial(2)   ← 結果待ち
              factorial(2)  →  2 * factorial(1)
                 factorial(1)  →  1  ← ベースケース (n <= 1)

  【帰りの計算（スタックを畳む）】
                 factorial(1)  戻り値 = 1
              factorial(2)     戻り値 = 2 * 1   = 2
           factorial(3)        戻り値 = 3 * 2   = 6
        factorial(4)           戻り値 = 4 * 6   = 24
     factorial(5)              戻り値 = 5 * 24  = 120

  ⚠ ベースケースがないと無限に積み上がる → "Maximum call stack size exceeded"

  ● ネスト構造のフラット化も同じ原理
    {a: 1, b: {c: 2, d: {e: 3}}}
      prefix='', key='a' → 'a'=1
      prefix='', key='b' → 中身がオブジェクト → flattenObj(中身, 'b')
        prefix='b', key='c' → 'b.c'=2
        prefix='b', key='d' → 中身オブジェクト → flattenObj(中身, 'b.d')
          prefix='b.d', key='e' → 'b.d.e'=3
    結果: { 'a':1, 'b.c':2, 'b.d.e':3 }
JavaScript
// 階乗（n!）
function factorial(n) {
  if (n <= 1) return 1; // ベースケース
  return n * factorial(n - 1);
}
console.log(`5! = ${factorial(5)}`);

// ネストオブジェクトのフラット化
function flattenObj(obj, prefix = '') {
  const result = {};
  for (const [k, v] of Object.entries(obj)) {
    const key = prefix ? `${prefix}.${k}` : k;
    if (v && typeof v === 'object' && !Array.isArray(v)) {
      Object.assign(result, flattenObj(v, key));
    } else {
      result[key] = v;
    }
  }
  return result;
}

const nested = { a: { b: { c: 1 }, d: 2 }, e: 3 };
console.log(JSON.stringify(flattenObj(nested)));

// 木構造の走査
const tree = {
  name: 'root',
  children: [
    { name: 'A', children: [{ name: 'A1', children: [] }] },
    { name: 'B', children: [] },
  ]
};
function walk(node, depth = 0) {
  console.log(`${'  '.repeat(depth)}└ ${node.name}`);
  node.children.forEach(c => walk(c, depth + 1));
}
walk(tree);
