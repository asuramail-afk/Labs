00

擬似要素とは何か（概念とコロン記法）

基本

擬似要素（pseudo-element）は HTMLには存在しないのに CSS で作れる「仮想の子要素」です。実 DOM には現れないのにページには描画されます。コロン2つ ::name で書き、擬似クラス :hover などとは文法が区別されます。

🔰 イメージ: 擬似要素は「見えない子供」
要素 <p> に対して ::before を書くと、ブラウザが描画時に p の先頭に仮想の小さな子要素をそっと挿入してくれる、と思えばOK。DevTools では確認できるが、HTMLソースには出てこない。 
HTML (書いたもの):
┌───────────────────┐
│ <p>本文</p>        │
└───────────────────┘

CSSで p::before { content:'★'; } と書くと...

ブラウザが描画:
┌────────────────────────────────┐
│ <p> [★::before] 本文 [::after] </p>
│       ↑ 仮想子要素      ↑ 仮想子要素
└────────────────────────────────┘
        DOMには存在しない / 画面には映る
分類記法代表例何を指すか
擬似要素::name::before / ::after / ::first-letter要素の一部（仮想の子）
擬似クラス:name:hover / :focus / :nth-child要素の状態や位置
✗ Before（CSS2 旧記法／混同しやすい）
CSS — 旧コロン1つ
/* 擬似要素なのにコロン1つ。動くが非推奨 */
p:before { content:'★'; }
p:hover  { color:red; }  /* これは擬似クラスで正しい */
✓ After（CSS3 推奨記法）
CSS — コロン2つで擬似要素を明示
/* コロン2つで「擬似要素」と一目で分かる */
p::before { content:'★'; }
p:hover   { color:red; }  /* 擬似クラスはコロン1つのまま */
01

::before / ::after — 要素の前後に仮想の子を挿入

基本

最もよく使う2つの擬似要素。::before は要素の内容の先頭、::after は末尾に仮想の子を挿入します。必ず content プロパティが必要（無いと描画されない）。下のライブデモは引用符を ::before/::after で装飾しています。

HTML + CSS — 引用文に上下のクォートを装飾
<!-- HTML -->
<blockquote class="quote">擬似要素は仮想の子供です</blockquote>

/* CSS */
.quote          { position:relative; padding:18px 40px; }
.quote::before { content:'"'; position:absolute;
                  left:8px; top:0; font-size:2.8rem; }
.quote::after  { content:'"'; position:absolute;
                  right:8px; bottom:-12px; }
現在: After（装飾ON）

擬似要素を使えば、HTMLを汚さずに引用符や装飾を加えられる。これが最大の強み。

02

content プロパティの全て

基本

content は擬似要素の描画内容を決める必須プロパティ。文字列、画像、属性値、カウンター、空文字など多彩。省略すると ::before/::after は生成すらされない（これが落とし穴）。

記法効果よく使う場面
content:'★'文字列を挿入アイコン・装飾
content:''空文字（要素は生成される）図形や線を描画する空箱
content:url(icon.svg)画像を挿入アイコン
content:attr(href)属性値を表示印刷時にURL表示
content:counter(sec)自動番号章番号・目次
content:none生成しないOFFにする
HTML + CSS — attr() で href を表示（印刷に便利）
<!-- HTML -->
<p class="attr-demo">
  詳しくは <a href="https://example.com">こちら</a> 参照
</p>

/* CSS: リンクの直後に (URL) を自動挿入 */
.attr-demo a::after {
  content: ' (' attr(href) ')';
  color: #5c6370;
  font-size: .8rem;
}
現在: After（content あり）

03

::first-letter — ドロップキャップ

中級

ブロック要素の最初の1文字だけを狙い撃ちできる擬似要素。雑誌風のドロップキャップが HTMLを汚さずに作れます。インライン要素には適用できません（display:block 系に限る）。

CSS — 先頭文字だけ巨大化
.paragraph::first-letter {
  font-size: 2.6rem;
  font-weight: 700;
  color: #005cc5;
  float: left;         /* 本文を回り込ませる */
  line-height: 1;
  padding: 4px 8px 0 0;
}
現在: After（::first-letter/::first-line ON）

擬似要素を使うと、先頭の1文字や最初の1行だけに特別な装飾を与えられます。これは HTMLタグを追加せずに達成できるので、本文の構造を汚しません。雑誌のような見栄えを簡単に作れます。

04

::first-line — 最初の「見えている1行」

中級

要素内の視覚的な最初の1行に適用される擬似要素。ウィンドウ幅で1行の長さが変わるため、ブラウザが動的に判定します。これが一番の見どころで、幅を変えると適用範囲も変化。

📘 ポイント: ::first-line は「HTML上の最初の行」ではなく「描画結果の1行目」を指します。ウィンドウを狭めると1行目も変わる。適用できるプロパティは限定的（font、color、background、text-decoration など）。
CSS — 1行目だけ緑の太字に
.paragraph::first-line {
  font-weight: 600;
  color: #22863a;
  letter-spacing: .02em;
}
05

::selection — 選択された範囲のスタイル

中級

ユーザがドラッグして選択したテキストの背景色・文字色を変えられる擬似要素。ブランドカラーに合わせた選択色を使うと「ちゃんと作られている感」が出ます。使えるプロパティは color / background-color / text-decoration / text-shadow などに限定。

CSS — 選択範囲を黄色×赤文字に
.selectable::selection {
  background: #ffeb3b;
  color: #cb2431;
}

▼ この文章をマウスでドラッグして選択してみてください。通常の青色ではなく、黄色い背景に赤い文字で選択範囲が表示されるはずです。ブランドカラーに合わせると良い印象を与えます。

06

::placeholder — 入力欄のヒント文字

基本

<input placeholder="..."> のヒント文字のスタイルを変える擬似要素。色だけでなくイタリック・文字間隔・透明度も変えられます。透明度 1 を明示しないと Firefox で薄く見える落とし穴あり。

HTML + CSS — プレースホルダを紫イタリックに
<!-- HTML -->
<input class="my-input" placeholder="メールアドレス">

/* CSS */
.my-input::placeholder {
  color: #6f42c1;
  font-style: italic;
  letter-spacing: .03em;
  opacity: .85;  /* Firefox は既定で薄い: 明示推奨 */
}
現在: After（紫イタリック）
07

::file-selector-button — ファイル選択ボタン

応用

<input type="file"> の「ファイルを選択」ボタンの見た目を変える擬似要素。これまでは ::-webkit-file-upload-button のベンダープレフィックス頼みで悩みの種でしたが、標準化されました（2022年ごろ全ブラウザ対応）。

CSS — ファイル選択ボタンを青い角丸に
.my-file::file-selector-button {
  background: #005cc5;
  color: #fff;
  border: none;
  padding: 8px 18px;
  border-radius: 6px;
  margin-right: 10px;
  cursor: pointer;
}
現在: After（青い角丸ボタン）
08

::marker — 箇条書きの「・」や数字

中級

<ul>/<ol>/<li> や <summary> のマーカー（黒丸・数字・▶︎）専用の擬似要素。これまでマーカー色だけ変えるのに list-style:none + ::before で偽装していた作業が1行で済むようになりました。

CSS — ulのマーカーを赤い▶に、olの数字を青太字に
.my-ul li::marker {
  color: #cb2431;
  font-weight: 800;
  content: '▶ ';   /* マーカー自体を置換 */
}
.my-ol li::marker {
  color: #005cc5;
  font-weight: 700;
}
現在: After（カスタムマーカー）
  • 最初の項目
  • 2番目の項目
  • 3番目の項目
  1. 手順1
  2. 手順2
  3. 手順3
09

::backdrop — ダイアログの背景

応用

<dialog> をモーダルで開いたときや全画面表示時に、ダイアログの後ろに敷かれる半透明レイヤーのスタイルを制御する擬似要素。これまで overlay用の div を自作していた作業が一気に不要に。

📘 発動条件: dialog.showModal() で開いた場合のみ。dialog.show() では backdrop は出ません。フルスクリーン API でも利用可能。
HTML + CSS — ダイアログの背景を青ぼかしに
<!-- HTML -->
<dialog id="myDialog">
  <p>これはモーダルダイアログです</p>
  <button onclick="myDialog.close()">閉じる</button>
</dialog>
<button onclick="myDialog.showModal()">開く</button>

/* CSS */
dialog::backdrop {
  background: rgba(0,92,197,.55);
  backdrop-filter: blur(3px);
}

これはモーダルダイアログです。背景の青いぼかしが ::backdrop で描画されています。
10

擬似要素 vs 擬似クラス — コロン数で使い分け

基本

初心者が混同しやすいのがこの2つ。コロンの数（: か ::）だけでなく、何を選んでいるかが本質的に違います。

🔰 覚え方
「要素の一部を作る／選ぶ」のが擬似要素（::）、「要素の状態や位置」を選ぶのが擬似クラス（:）。 
擬似要素 (::)           擬似クラス (:)
─────────────           ──────────────
p::before    ← 作る    a:hover      ← 状態
p::first-line ← 一部   li:first-child ← 位置
p::marker    ← 一部   input:focus   ← 状態
p::selection ← 一部   p:nth-of-type(2) ← 位置

本質: 要素を「分解/新設」 vs 要素を「状態で絞る」
目的書き方種類
カード先頭にアイコン挿入.card::before擬似要素
ホバー時だけ色変更.card:hover擬似クラス
1文字目だけ装飾p::first-letter擬似要素
2番目の子だけ装飾li:nth-child(2)擬似クラス
input のヒント文字input::placeholder擬似要素
入力欄にフォーカス中input:focus擬似クラス
✗ Before（よくある混同）
CSS — 意図不明
/* ::hover と書いてしまう */
a::hover { color:red; }  /* 動かない(擬似要素ではない) */

/* :before と書くのは旧記法で非推奨 */
p:before { content:'★'; } /* 動くがモダンではない */
✓ After（正しい使い分け）
CSS — 意図が明確
/* 擬似クラス = コロン1つ */
a:hover  { color:red; }

/* 擬似要素 = コロン2つ */
p::before { content:'★'; }

実践パターン集 — アイコン/必須マーク/外部リンク

応用

現場で頻繁に使う5つの ::before/::after レシピをまとめて確認します。HTML を変更せず CSS だけで装飾できるため、既存コンポーネントの拡張に強い。

CSS — 5つの実践パターン
/* 1. リンクアイコン */
.icon-link::before     { content:'🔗 '; }

/* 2. 外部リンクマーク */
.icon-ext::after       { content:' ↗'; color:#005cc5; }

/* 3. 必須フィールドの * */
.required::after       { content:' *'; color:#cb2431; }

/* 4. チェックマーク箇条書き（list-style不要） */
.checklist li::before  { content:'✓ '; color:#22863a; font-weight:800; }

/* 5. 区切り記号 */
.breadcrumb li + li::before { content:' / '; color:#999; }

関連ドキュメント / 外部サイト /

  • 実機で動作確認済み
  • HTMLを汚さない装飾
  • 保守性が高い
12

擬似要素の限界と注意点

応用

便利な擬似要素ですが「できないこと」も明確に知っておくと事故を減らせます。以下は現場でよく遭遇する5つの注意点。

できないこと理由 / 回避策
<img>/<input>/<br> に ::before/::afterこれらは「置換要素」「空要素」で子を持てない → ラッパdivに付ける
JSで DOM を取得::before は DOM に存在しない → querySelector で掴めない。getComputedStyle(el, '::before') で読み取りのみ可
複数の擬似要素を重ねる1要素につき ::before ::after 各1つだけ → 多層装飾にはラッパを増やす or background レイヤを併用
content なし で ::before生成されない → content:'' を必ず書く（空でもOK）
SEO/スクリーンリーダでの内容読み上げcontent の文字は読み上げ対応がブラウザ依存 → 重要テキストは擬似要素に入れない
⚠️ アクセシビリティ落とし穴: content:'必須' のような意味を持つ文字列を擬似要素に書くと、支援技術が読まない可能性があります。必須マークなら aria-required="true" など HTML 属性で意味を伝え、::after は視覚装飾のみに留めるのが安全。
✗ Before（img に ::after が効かない）
CSS — 置換要素に擬似要素
/* img は子を持てないので無視される */
img::after {
  content: 'NEW';  /* 表示されない */
}
✓ After（wrapper に付ける）
HTML + CSS — div で包む
<!-- HTML -->
<div class="img-wrap">
  <img src="photo.jpg">
</div>

/* CSS */
.img-wrap        { position:relative; }
.img-wrap::after { content:'NEW';
                     position:absolute;
                     top:8px; right:8px; }

wrapper div に付けた ::before（右上のリボン）と ::after（下の線）の両立例。置換要素では不可能だが、包めば自由。
