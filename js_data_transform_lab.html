配列 → 辞書オブジェクト（ルックアップ化）

基本

配列を id をキーにしたオブジェクトに変換。reduce と Object.fromEntries の2通り。 APIから取得したリストを「IDで即引き」できる辞書にする実務パターン。

🔰 なぜ辞書化が必要？
配列のままだと users.find(u => u.id === 3) で毎回ループが走る。辞書にすれば dict[3] で O(1) でアクセスできる。 
配列 (毎回探索)          辞書 (即アクセス)
┌───┬───┬───┐       ┌─────────────────┐
│ 0 │ 1 │ 2 │  →    │ "1": {id:1,...}  │
│{} │{} │{} │       │ "2": {id:2,...}  │
└───┴───┴───┘       │ "3": {id:3,...}  │
  find(id===3)        └─────────────────┘
  → 最大3回            dict[3] → 1回
reduce版
const users = [
  { id: 1, name: "Alice" },
  { id: 2, name: "Bob" },
  { id: 3, name: "Carol" }
];

// reduce版: 初期値 {} に1つずつ追加
const dict1 = users.reduce((acc, u) => {
  acc[u.id] = u;
  return acc;
}, {});

// fromEntries版: [key, value] のペア配列から一発変換
const dict2 = Object.fromEntries(
  users.map(u => [u.id, u])
);

console.log("reduce版:", dict1);
console.log("fromEntries版:", dict2);
console.log("ID=2を即取得:", dict1[2]);

配列 → ツリー構造復元

応用

DBから取得したフラットな親子レコードを、children プロパティ付きのツリーに復元する。 カテゴリツリーやファイルツリーの構築で頻出。

🔰 変換のイメージ
フラット配列（DBの行データ）       ツリー構造
┌──────────────────────┐      root (id:1)
│ {id:1, parent:null}  │  →    ├── child (id:2)
│ {id:2, parent:1   }  │       └── child (id:3)
│ {id:3, parent:1   }  │            └── grandchild (id:4)
│ {id:4, parent:3   }  │
└──────────────────────┘
フラット→ツリー変換
const rows = [
  { id: 1, parent: null, name: "ルート" },
  { id: 2, parent: 1,    name: "子A" },
  { id: 3, parent: 1,    name: "子B" },
  { id: 4, parent: 3,    name: "孫B-1" }
];

function buildTree(items) {
  const map = {};
  const roots = [];

  // Pass 1: 全ノードを辞書化 + children初期化
  items.forEach(item => {
    map[item.id] = { ...item, children: [] };
  });

  // Pass 2: 親子関係をつなぐ
  items.forEach(item => {
    if (item.parent === null) {
      roots.push(map[item.id]);
    } else {
      map[item.parent].children.push(map[item.id]);
    }
  });
  return roots;
}

const tree = buildTree(rows);
console.log(JSON.stringify(tree, null, 2));

基本

集約パターン 4種
const orders = [
  { product: "A", price: 100 },
  { product: "B", price: 250 },
  { product: "A", price: 100 },
  { product: "C", price: 80 },
  { product: "B", price: 250 },

// 合計

配列 → 配列（形状変換）

基本

flat（フラット化）、chunk（チャンク分割）、zip（2配列合成）の3つ。配列が「形は変わるが配列のまま」のパターン。

flat / chunk / zip
// ① flat: ネスト配列をフラット化
const nested = [[1,2],[3,[4,5]]];

// ② chunk: 配列をN個ずつ分割
function chunk(arr, size) {
  const result = [];
  for (let i = 0; i < arr.length; i += size) {
    result.push(arr.slice(i, i + size));
  }
  return result;
}

// ③ zip: 2つの配列をペアにまとめる

pick / omit（部分抽出）

基本

必要なプロパティだけ取り出す pick と、不要なプロパティを除外する omit。 APIレスポンスからUIに必要な項目だけを取り出すときに頻出。

pick / omit 自作
const user = {
  id: 1, name: "Tom", email: "tom@ex.com",
  password: "secret", role: "admin"
};

// pick: 必要なキーだけ取り出す
function pick(obj, keys) {
  return Object.fromEntries(
    keys.filter(k => k in obj).map(k => [k, obj[k]])
  );
}

// omit: 不要なキーを除外
function omit(obj, keys) {
  const excluded = new Set(keys);
  return Object.fromEntries(
    Object.entries(obj).filter(([k]) => !excluded.has(k))
  );
}

console.log("pick:", pick(user, ["name", "email"]));
console.log("omit:", omit(user, ["password"]));

ネスト ↔ フラット変換

応用

深いネストオブジェクトを "user.addr.city" のようなドット区切りキーに展開（flatten）し、逆にドット区切りからネスト復元（unflatten）する。

flatten / unflatten
// flatten: ネスト → ドットキー
function flatten(obj, prefix = "") {
  const result = {};
  for (const [k, v] of Object.entries(obj)) {
    const key = prefix ? `${prefix}.${k}` : k;
    if (v && typeof v === "object" && !Array.isArray(v)) {
      Object.assign(result, flatten(v, key));
    } else {
      result[key] = v;
    }
  }
  return result;
}

// unflatten: ドットキー → ネスト
function unflatten(obj) {
  const result = {};
  for (const [k, v] of Object.entries(obj)) {
    const parts = k.split(".");
    let cur = result;
    parts.forEach((p, i) => {
      if (i === parts.length - 1) { cur[p] = v; }
      else { cur[p] = cur[p] || {}; cur = cur[p]; }
    });
  }
  return result;
}

const nested = { user: { name: "Tom", addr: { city: "Osaka", zip: "530" } } };
const flatObj = flatten(nested);
console.log("flatten:", flatObj);
console.log("unflatten:", unflatten(flatObj));

浅いマージ vs 深いマージ

応用

Object.assign / スプレッド（浅い）と再帰的マージ（深い）の違いを実行して確認。

shallow vs deep merge
const base = { a: 1, b: { x: 10, y: 20 } };
const patch = { b: { y: 99, z: 30 }, c: 3 };

// 浅いマージ: b が丸ごと上書きされる
const shallow = { ...base, ...patch };
console.log("浅いマージ:", JSON.stringify(shallow));
// → b: {y:99, z:30}  ← x:10 が消える！

// 深いマージ: ネスト内も統合
function deepMerge(target, source) {
  const result = { ...target };
  for (const [k, v] of Object.entries(source)) {
    if (v && typeof v === "object" && !Array.isArray(v)
        && target[k] && typeof target[k] === "object") {
      result[k] = deepMerge(target[k], v);
    } else {
      result[k] = v;
    }
  }
  return result;
}

const deep = deepMerge(base, patch);
console.log("深いマージ:", JSON.stringify(deep));
// → b: {x:10, y:99, z:30}  ← x:10 が残る

ツリー → フラット配列

応用

Lab03の逆方向。ツリー構造を再帰で走査してフラット配列に戻す。depth情報を付与するパターンも。

tree → flat (with depth)
const tree = {
  id: 1, name: "root", children: [
    { id: 2, name: "childA", children: [] },
    { id: 3, name: "childB", children: [
      { id: 4, name: "grandchild", children: [] }
    ]}
  ]
};

function flattenTree(node, depth = 0) {
  const { children, ...rest } = node;
  return [
    { ...rest, depth },
    ...children.flatMap(c => flattenTree(c, depth + 1))
  ];
}

const flat = flattenTree(tree);
console.log("フラット化:");
flat.forEach(n =>
  console.log(`${"  ".repeat(n.depth)}${n.name} (depth:${n.depth})`)
);

DOM → JSON ツリー

応用

実際のDOM要素を再帰的に走査して仮想DOM風のJSONに変換する。React/OutSystemsの内部で起きていることの理解に直結。

HelloWorld
DOM → JSON 変換
function domToJson(el) {
  if (el.nodeType === Node.TEXT_NODE) {
    return el.textContent.trim() || null;
  }
  return {
    tag: el.tagName.toLowerCase(),
    attrs: Object.fromEntries(
      [...el.attributes].map(a => [a.name, a.value])
    ),
    children: [...el.childNodes]
      .map(domToJson)
      .filter(c => c !== null)
  };
}

CSV文字列 → オブジェクト配列

基本

CSV文字列を1行目ヘッダー+残りデータとして、構造化されたオブジェクト配列に変換する。

CSV → Array of Objects
const csv = `name,age,city
Tom,30,Osaka
Alice,25,Tokyo
Bob,35,Nagoya`;

function parseCSV(text) {
  const lines = text.trim().split("\n");
  const headers = lines[0].split(",");
  return lines.slice(1).map(line => {
    const vals = line.split(",");
    return Object.fromEntries(
      headers.map((h, i) => [h, vals[i]])
    );
  });
}

const data = parseCSV(csv);
console.log("パース結果:", data);
console.log("2番目:", data[1]);

APIレスポンス正規化

応用

バックエンドから返ってきたsnake_caseのレスポンスを、フロントエンドのcamelCase＋UI用フラグ付きの形に変換する。

🔰 なぜ正規化が必要？
バックエンド（Python/Ruby）は snake_case、フロント（JS）は camelCase が慣習。 さらにUI表示用のフラグ（hasMore 等）を追加で算出する。 
API Response               Frontend State
{                          {
  "total_count": 85,   →     totalCount: 85,
  "next_page": 3,             nextPage: 3,
  "data": [...]               items: [...],
}                              hasMore: true  ← 算出
                            }
API → フロント変換
// バックエンドのレスポンス (snake_case)
const apiResponse = {
  total_count: 85,
  current_page: 2,
  per_page: 20,
  data: [
    { user_id: 1, first_name: "Tom", is_active: true },
    { user_id: 2, first_name: "Ann", is_active: false }
  ]
};

// snake_case → camelCase 変換
function toCamel(s) {
  return s.replace(/_([a-z])/g, (_, c) => c.toUpperCase());
}
function camelizeKeys(obj) {
  if (Array.isArray(obj)) return obj.map(camelizeKeys);
  if (obj && typeof obj === "object") {
    return Object.fromEntries(
      Object.entries(obj).map(([k, v]) => [toCamel(k), camelizeKeys(v)])
    );
  }
  return obj;
}

const camelized = camelizeKeys(apiResponse);

// UI用のフラグを追加
const uiState = {
  items: camelized.data,
  totalCount: camelized.totalCount,
  currentPage: camelized.currentPage,
  totalPages: Math.ceil(camelized.totalCount / camelized.perPage),

};

正規化（normalize）と逆正規化

上級

ネストしたAPIレスポンスをエンティティごとのフラットなテーブルに分解（正規化）し、表示時に再組み立て（逆正規化 / denormalize）する。Redux等の状態管理で必須。

🔰 正規化のメリット
同じ author が複数 post に紐づくとき、ネストしたままだと author 情報が重複する。 正規化すれば author は1箇所に保持され、更新が1回で済む。 
ネスト（重複あり）              正規化（重複なし）
{posts: [                     posts:   {1: {authorId:10,...}}
  {id:1, author:{id:10,...}},  authors: {10: {name:"Tom"}}
  {id:2, author:{id:10,...}}   ← 10番は1箇所だけ！
]}  ← 10番が2回出現
normalize / denormalize
// APIからのネストしたレスポンス
const apiData = {
  posts: [
    { id: 1, title: "Hello",
      author: { id: 10, name: "Tom" },
      comments: [
        { id: 100, text: "Great!" },
        { id: 101, text: "Thanks" }
      ]},
    { id: 2, title: "World",
      author: { id: 10, name: "Tom" },
      comments: [
        { id: 102, text: "Nice" }
      ]}
  ]
};

// ── 正規化 ──
function normalize(data) {
  const store = { posts: {}, authors: {}, comments: {} };
  data.posts.forEach(post => {
    store.authors[post.author.id] = post.author;
    post.comments.forEach(c => { store.comments[c.id] = c; });
    store.posts[post.id] = {
      id: post.id,
      title: post.title,
      authorId: post.author.id,
      commentIds: post.comments.map(c => c.id)
    };
  });
  return store;
}

// ── 逆正規化（表示用に再組み立て）──
function denormalize(postId, store) {
  const post = store.posts[postId];
  return {
    ...post,
    author: store.authors[post.authorId],
    comments: post.commentIds.map(id => store.comments[id])
  };
}

const store = normalize(apiData);
console.log("=== 正規化後 ===");
console.log(JSON.stringify(store, null, 2));

console.log("=== 逆正規化（post 1）===");
console.log(JSON.stringify(denormalize(1, store), null, 2));
