User-Agent → Client Hints への移行

User-Agent → Client Hints への移行: 従来（User-Agent）: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/124.0.0.0 Safari/537.36 → 1行に全情報、パース困難、フィンガープリントに悪用されやすい 新方式（Client Hints）: 低エントロピー（デフォルト送信、フィンガープリントリスク低）: Sec-CH-UA: "Chromium";v="124", "Google Chrome";v="124" Sec-CH-UA-Mobile: ?0 Sec-CH-UA-Platform: "Windows" 高エントロピー（サーバーがオプトインして初めて送信）: サーバー: Accept-CH: Sec-CH-UA-Platform-Version, Sec-CH-UA-Model ↓ 次のリクエストから追加送信 Sec-CH-UA-Platform-Version: "15.0.0" Sec-CH-UA-Full-Version-List: "Chromium";v="124.0.6367.60", ...

低エントロピー（自動送信）

Sec-CH-UA
  ブラウザブランドとメジャーバージョン
  例: "Chromium";v="124", "Google Chrome";v="124"

Sec-CH-UA-Mobile
  モバイルデバイスか否か
  例: ?0（非モバイル）/ ?1（モバイル）

Sec-CH-UA-Platform
  OS名（大まかな情報のみ）
  例: "Windows" / "macOS" / "Android"

高エントロピー（Accept-CHで要求）

Sec-CH-UA-Platform-Version
  OSの詳細バージョン
  例: "15.0.0"

Sec-CH-UA-Model
  デバイスモデル（モバイル向け）
  例: "Pixel 7"

Sec-CH-UA-Full-Version-List
  全ブランドの完全バージョン

Sec-CH-UA-Arch
  CPUアーキテクチャ: "x86" / "arm"

Sec-CH-UA-Bitness
  64ビット / 32ビット
なぜ User-Agent を廃止するのか？
User-Agent 文字列は1行に OS・ブラウザ・エンジン・バージョンが詰め込まれており、 パースが困難なうえ、ブラウザごとに形式が異なります。また、すべての情報がデフォルトで送信されるため、 フィンガープリント（ユーザー追跡）に悪用されやすい問題があります。 Chrome は段階的に User-Agent を「凍結（freeze）」し、Client Hints への移行を進めています。

User-Agent の「凍結」とは？
Google Chrome は段階的に User-Agent 文字列の情報を削減（凍結）しています。 将来的には Chrome/XX.0.0.0 のようにマイナーバージョンが固定され、 OS 情報も一部が削除されます。詳細なブラウザ情報は Sec-CH-UA-* ヘッダー経由でのみ取得可能になります。

実験 12-D: 高エントロピーヒントの取得

navigator.userAgentData.getHighEntropyValues() を使うと、 JavaScriptから高エントロピーヒントを取得できます。これはサーバー側の Accept-CH とは独立した、 クライアントサイドのAPIです。

サーバーサイド（Accept-CH） vs クライアントサイド（getHighEntropyValues）: サーバーサイド（Accept-CH）: サーバー側レスポンスヘッダー: Accept-CH: Sec-CH-UA-Platform-Version, Sec-CH-UA-Model ↓ 次のリクエストから自動送信 Sec-CH-UA-Platform-Version: "15.0.0" クライアントサイド（JS API）: navigator.userAgentData.getHighEntropyValues(['platformVersion']) → Promise を返す → .then(values => values.platformVersion) → ネットワークリクエストなし、JS内で直接取得

Accept-CH はどこで設定する？
サーバー側の高エントロピーオプトインは、HTTP レスポンスヘッダーに Accept-CH: Sec-CH-UA-Platform-Version, Sec-CH-UA-Model を付与することで行います。 OutSystems の場合、REST API の OnAfterResponse で SetResponseHeader を使って設定できます。 ブラウザは次のリクエストから該当のヒントを自動送信するようになります。
