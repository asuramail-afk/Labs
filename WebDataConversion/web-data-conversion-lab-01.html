🧪 Lab 01: URL・ドメイン関連の変換

Web Data Conversion Lab シリーズ

ブラウザはURLを通じてサーバーと通信します。しかしURLには使用できない文字（日本語・記号）があり、 ブラウザは送信前にさまざまな変換を自動で行っています。このLabではその変換をJavaScriptで再現・確認します。

⏱ 所要時間：20分 難易度：★☆☆

📋 前提知識：なし（シリーズの出発点）

🎯 このLabの学習目標

完了すると、以下ができるようになります：

1. encodeURIComponent で日本語・記号がパーセントエンコードされる仕組みを理解する
2. encodeURI と encodeURIComponent の違いを説明できる
3. 日本語ドメインが Punycode で自動変換されることを確認する
4. ../ や ./ を含むパスが 正規化される仕組みを理解する
5. URL全体の構造（scheme / host / path / query / fragment）を分解できる

実験 1-A: URLエンコード（encodeURIComponent）

URLにはASCII文字以外（日本語、スペース、一部記号）をそのまま含めることができません。
encodeURIComponent() はこれらの文字を %XX 形式（パーセントエンコーディング）に変換します。

STEP 1 — 文字列をエンコードしてみる

下の入力欄にテキストを入力すると、リアルタイムでエンコード結果が表示されます。

STEP 2 — いろいろな文字で試す

ボタンを押すと、さまざまなパターンの文字列が入力されます。

観察ポイント: 英数字（A-Z a-z 0-9）と - _ . ~ ! * ' ( ) はエンコードされません。 日本語は1文字が 3つの %XX に変換されます（UTF-8で3バイトだから）。

逆変換: decodeURIComponent()

よくある落とし穴: 日本語をそのままURLに入れて404エラーになる
正しい使い方: "?q=" + encodeURIComponent(keyword)

実験 1-B: encodeURI と encodeURIComponent の違い

似た名前の2つの関数ですが、何をエンコードするかが異なります。 間違えるとURLが壊れたり、パラメータが分離したりします。

STEP 1 — 同じ入力で両方を比較

encodeURI（URL全体用）

← 比較実行を押してください

encodeURIComponent（パーツ用）

← 比較実行を押してください
STEP 2 — 正しい使い分け
// ❌ 間違い: パラメータ値に encodeURI
const query = encodeURI("price=100&tax=10");
// → "price=100&tax=10"  ← & がそのまま！パラメータが分離する

// ✅ 正解: パラメータ値に encodeURIComponent
const query = encodeURIComponent("price=100&tax=10");
// → "price%3D100%26tax%3D10"  ← 安全に1つの値として扱われる

// 実用例: 検索URLの組み立て
const keyword = "JS 基礎";
const url = "https://example.com/search?q=" + encodeURIComponent(keyword);
// → "https://example.com/search?q=JS%20%E5%9F%BA%E7%A4%8E"
STEP 3 — URLSearchParams との挙動差（実務で頻出）

URLSearchParams もクエリ文字列をエンコードしますが、 スペースの扱いが異なります。この差を知らないとバグの原因になります。 

← ボタンを押すと比較結果が表示されます
使い分けルール: encodeURI = URL全体をエンコード（構造文字は保持）、 encodeURIComponent = URLの部品（パラメータの値など）をエンコード、 URLSearchParams = クエリ文字列の組み立てに最適（スペースは + になる）。
迷ったら URLSearchParams でクエリを組み立て、それ以外は encodeURIComponent。

実験 1-C: Punycode（日本語ドメインの変換）

ドメイン名（example.com の部分）は英数字とハイフンしか使えません。 日本語ドメインは Punycode（xn-- で始まる形式）に自動変換されます。 通常はブラウザが自動で行いますが、URL オブジェクトで再現できます。

STEP 1 — 日本語ドメインをPunycodeに変換
← 変換ボタンを押してください
STEP 2 — いろいろなドメインで試す
なぜPunycodeが必要？: DNS（ドメインネームシステム）はASCII文字しか扱えません。 ブラウザのアドレスバーでは日本語が見えていても、実際の通信では xn-- 形式に変換されています。
📖 参考: MDN — 国際化ドメイン名 (IDN)
実行者: ブラウザが自動で実行
タイミング: DNS解決前
逆変換: ブラウザがアドレスバーで自動復元
📋 初心者まとめ
覚える関数: new URL("http://"+domain).hostname
よくある落とし穴: 日本語ドメインをそのままDNS問い合わせして名前解決失敗
正しい使い方: URLオブジェクト生成で自動的にPunycodeへ変換される

実験 1-D: パス正規化（Path Normalization）

URLのパスに ../（上のディレクトリ）や ./（同じディレクトリ）が含まれると、 ブラウザはリクエスト送信前に正規化（階層計算して整理）します。

STEP 1 — パスを正規化してみる
← 正規化ボタンを押してください
STEP 2 — いろいろなパスで試す
セキュリティとの関連: 最後のボタン /../../etc/passwd はパストラバーサル攻撃の例です。 ブラウザの正規化はルート（/）を超えないため /etc/passwd に整理されます。 サーバー側でも必ずパスのバリデーションを行う必要があります。
ポイント: パス正規化はブラウザが自動で行います。 ../ が解決された後のパスがサーバーに送られるため、サーバーは正規化済みのパスを受け取ります。
📖 参考: MDN — URL API
実行者: ブラウザが自動で実行
タイミング: リクエスト送信前
逆変換: なし（一方向の整理）
📋 初心者まとめ
覚える関数: new URL(path, base).pathname
よくある落とし穴: ../を含むパスを文字列結合だけで処理してディレクトリトラバーサルが発生
正しい使い方: URLオブジェクトで正規化 + サーバー側でも必ずパス検証を実施

実験 1-E: URL構造の完全分解

URL オブジェクトを使うと、URLのあらゆる部品を個別に取り出せます。 手動でのパース（split('/') や正規表現）は不要です。

/* --- Lab 01: URL・ドメイン関連の全変換 --- */

// ① URLエンコード
const rawText = "JS 基礎";
const urlEncoded = encodeURIComponent(rawText);
console.log("URLエンコード:", rawText, "→", urlEncoded);

// ② Punycode (ドメインの変換)
const domain = "総務省.jp";
const punyUrl = new URL(`http://${domain}`);
console.log("Punycode:", domain, "→", punyUrl.hostname);

// ③ パス正規化
const rawPath = "/a/b/../c/./d";
const normalizedPath = new URL(rawPath, "http://e.com").pathname;
console.log("パス正規化:", rawPath, "→", normalizedPath);

// ④ URLSearchParams（スペースの扱いに注目）
const uspResult = new URLSearchParams({q: rawText}).toString();
console.log("URLSearchParams:", rawText, "→", uspResult);

// ── 全変換サマリ（console.table で一覧表示）──
console.table([

  { 変換: "URLSearchParams", 入力: rawText, 出力: uspResult,      実行者: "開発者" },
  { 変換: "Punycode",        入力: domain,  出力: punyUrl.hostname, 実行者: "ブラウザ自動" },
  { 変換: "パス正規化",       入力: rawPath, 出力: normalizedPath,  実行者: "ブラウザ自動" },

まとめ: この3つの変換は、ブラウザがリクエスト送信前に自動で行っています。 普段意識しませんが、JavaScriptの URL オブジェクトと encodeURIComponent を使えば、 同じ処理を明示的に再現・確認できます。
