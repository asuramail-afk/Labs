実験 7-A: response.json() / response.text() / response.blob()

🔰 初心者向け解説

fetch()で受け取る「荷物（Response）」は、中身を取り出す方法が3種類あります。

  • テキストとして読む → response.text()：手紙を読む
  • JSONとして解析する → response.json()：翻訳済みの資料として読む
  • 生データとして保存する → response.blob()：荷物を箱ごと保管する

重要なルール：荷物の中身は1回しか取り出せません。2回読みたいときは response.clone() で複製を作ります。

Step 1: JSON取得 — .json() / .text() / .blob() の比較

STEP 1 — response.clone() を使って同じレスポンスを複数回読む

httpbin.org/json から取得して3つの方法で読み比べます：

// レスポンスボディは1回しか読めないため clone() が必要
const res = await fetch('https://httpbin.org/json');
const res2 = res.clone(); // ← 2回読むために複製
const res3 = res.clone(); // さらに複製

const asJson = await res.json();   // JavaScriptオブジェクトに変換
const asText = await res2.text();  // 文字列としてそのまま取得
const asBlob = await res3.blob();  // Blobオブジェクトとして取得

console.log(typeof asJson);  // "object"
console.log(typeof asText);  // "string"
console.log(asBlob.type);    // "application/json"
console.log(asBlob.size);    // バイト数
ボタンをクリックすると実行されます

Step 2: 画像取得 — response.blob() で画像データを受け取る

STEP 2 — httpbin.org/image/png から PNG を Blob として取得
// バイナリデータ（画像）を取得
const res = await fetch('https://httpbin.org/image/png');
const blob = await res.blob();

console.log(blob.type);  // "image/png"
console.log(blob.size);  // バイト数（PNG全体）

// Blob URL を作成して <img> に表示
const url = URL.createObjectURL(blob);
const img = document.createElement('img');
img.src = url;
ボタンをクリックすると実行されます

Step 3: BodyUsedエラー体験

STEP 3 — clone() なしで2回読もうとするとエラーになる
// 同じレスポンスを clone なしで2回読む → エラー
const res = await fetch('https://httpbin.org/json');
await res.json();  // 1回目: OK
await res.json();  // 2回目: TypeError: body stream is disturbed or locked

// res.bodyUsed が true になっていることを確認
console.log(res.bodyUsed); // true（読み終わり済み）
ボタンをクリックすると実行されます
✅ 確認チェックリスト
response.json() と response.text() で受け取る型の違いを確認した
response.blob() で画像データを取得し、Blob URLで表示できた
response.clone() なしで2回読むとエラーになることを体験した
📋 初心者まとめ
覚える関数: response.json() / .text() / .blob()
よくある落とし穴: responseを2回読もうとしてTypeError: body already consumed（bodyは一度しか読めない）
正しい使い方: 複数回読む場合は const r2 = response.clone() してから各メソッドを呼ぶ

実験 7-B: TextDecoder — 文字コードの逆変換

🔰 初心者向け解説

コンピュータ内部では文字はバイト列（数値の並び）で保存されています。「外国語の手紙を正しい辞書で翻訳する」ようなイメージです。

  • UTF-8の「あ」= E3 81 82（3バイト）
  • Shift_JISの「あ」= 82 A0（2バイト）
  • 辞書（デコーダー）を間違えると → 文字化け！

Step 1: UTF-8デコード

STEP 1 — UTF-8バイト列を TextDecoder で文字列に変換
// UTF-8の「こんにちは」のバイト列
const bytes = new Uint8Array([
  0xe3,0x81,0x93, // こ
  0xe3,0x82,0x93, // ん
  0xe3,0x81,0xab, // に
  0xe3,0x81,0xa1, // ち
  0xe3,0x81,0xaf  // は
]);

const decoder = new TextDecoder('utf-8');
const text = decoder.decode(bytes);
console.log(text); // "こんにちは"
ボタンをクリックすると実行されます

Step 2: 任意テキストのエンコード→デコード往復

STEP 2 — TextEncoder でバイト列に変換し、TextDecoder で戻す
// 送信側: TextEncoder でエンコード
const encoder = new TextEncoder(); // 常にUTF-8
const bytes = encoder.encode('Hello, 世界！');

// 受信側: TextDecoder でデコード
const decoder = new TextDecoder('utf-8');
const text = decoder.decode(bytes);
console.log(text); // "Hello, 世界！"
ボタンをクリックすると実行されます

Step 3: 文字コードを間違えた場合の文字化け体験

STEP 3 — UTF-8バイト列をShift_JISで読む → 文字化け
// UTF-8の「日本語」のバイト列
const utf8Bytes = new TextEncoder().encode('日本語');

// 正しい辞書（UTF-8）で読む
const correct = new TextDecoder('utf-8').decode(utf8Bytes);
console.log(correct); // "日本語"

// 間違った辞書（Shift_JIS）で読む → 文字化け
const garbled = new TextDecoder('shift-jis', { fatal: false }).decode(utf8Bytes);
console.log(garbled); // "???"（文字化け）
ボタンをクリックすると実行されます
TextDecoderはW3Cの Encoding API で定義されており、UTF-8・Shift_JIS・EUC-JP・UTF-16LE/BEなど多くの文字コードをサポートします。modern fetchでは基本的にUTF-8が使われます。
✅ 確認チェックリスト
UTF-8バイト列を TextDecoder で文字列にデコードできた
TextEncoder → TextDecoder の往復で元の文字列が戻ることを確認した
文字コードを間違えると文字化けが起きることを体験した
📖 文字コード判定カスケード
ブラウザはHTML文書の文字コードを以下の優先順位で決定します:
① HTTP ヘッダー（Content-Type: charset=utf-8）→ ② BOM（ファイル先頭のバイト列）→ ③ <meta charset>（HTML内の宣言）→ ④ ブラウザの自動推測（統計的手法）→ ⑤ デフォルト（windows-1252等）
ベストプラクティス: 必ずHTTPヘッダーで charset=utf-8 を返し、HTMLの先頭に <meta charset="utf-8"> を配置する

実験 7-C: atob() — Base64デコード

🔰 初心者向け解説

Base64は「バイナリデータをテキストに変換する」エンコード方式です。atob() はその逆変換です。

  • btoa('Hello') → "SGVsbG8="（圧縮ファイルに梱包）
  • atob('SGVsbG8=') → "Hello"（圧縮ファイルを解凍）
  • 日本語は直接 atob() で扱えない → atob → Uint8Array → TextDecoder の手順が必要

Step 1: ASCII文字のBase64デコード

STEP 1 — btoa() でエンコードして atob() でデコード
// エンコード（送信側）
const original = 'Hello, Base64!';
const encoded = btoa(original);
console.log(encoded); // "SGVsbG8sIEJhc2U2NCE="

// デコード（受信側）
const decoded = atob(encoded);
console.log(decoded); // "Hello, Base64!"
ボタンをクリックすると実行されます

Step 2: 日本語のBase64デコード（正しい手順）

STEP 2 — 日本語Base64の正しいデコード手順
// 送信側: 日本語 → UTF-8バイト列 → Base64
const text = '受信側の逆変換';
const utf8 = new TextEncoder().encode(text);
const b64 = btoa(String.fromCharCode(...utf8));

// 受信側: Base64 → バイト列 → TextDecoder（UTF-8）
const raw = atob(b64);                // バイナリ文字列
const bytes = Uint8Array.from(raw, c => c.charCodeAt(0));
const result = new TextDecoder('utf-8').decode(bytes);
console.log(result); // "受信側の逆変換"
ボタンをクリックすると実行されます

Step 3: Data URLからBlobへの変換

STEP 3 — fetch(dataUrl).then(r => r.blob()) でData URLをBlobに変換
// Data URL（1×1の白ピクセルPNG）
const dataUrl = 'data:image/png;base64,' +
  'iVBORw0KGgoAAAANSUhEUgAAAAEAAAABCAYAAAAfFcSJAAAADUlEQVR42mNk+M9QDwADhgGAWjR9awAAAABJRU5ErkJggg==';

// fetch() に Data URL を渡すと Blob として取得できる
const blob = await fetch(dataUrl).then(r => r.blob());
console.log(blob.type); // "image/png"
console.log(blob.size); // バイト数
ボタンをクリックすると実行されます
atob() は直接マルチバイト文字を扱えません。日本語などを含む場合は必ず「atob → Uint8Array → TextDecoder('utf-8')」の3ステップで復元してください。
✅ 確認チェックリスト
ASCII文字を btoa() / atob() で往復できた
日本語の正しいデコード手順（atob → Uint8Array → TextDecoder）を実行できた
Data URLを fetch(dataUrl).then(r => r.blob()) でBlobに変換できた

実験 7-D: DecompressionStream — 圧縮データの展開

🔰 初心者向け解説

Gzip圧縮されたデータを展開するのは「圧縮袋を開けて元の大きさに戻す」イメージです。

  • 送信側: CompressionStream('gzip') → データを圧縮（小さくする）
  • 受信側: DecompressionStream('gzip') → データを展開（元に戻す）
  • 圧縮率はデータの種類による。テキストは高圧縮、すでに圧縮済みのデータ（PNG/ZIP等）は効果小

Step 1: テキストを圧縮して展開

STEP 1 — CompressionStream → DecompressionStream の往復
// 圧縮（送信側）
const text = '圧縮テストデータ。繰り返しが多いと効率が高い。'.repeat(10);
const bytes = new TextEncoder().encode(text);
const cs = new CompressionStream('gzip');
const writer = cs.writable.getWriter();
writer.write(bytes);
writer.close();
const compressed = await new Response(cs.readable).arrayBuffer();

// 展開（受信側）
const ds = new DecompressionStream('gzip');
const writer2 = ds.writable.getWriter();
writer2.write(compressed);
writer2.close();
const decompressed = await new Response(ds.readable).arrayBuffer();
const result = new TextDecoder().decode(decompressed);

console.log('元:', bytes.byteLength, 'bytes');
console.log('圧縮後:', compressed.byteLength, 'bytes');
console.log('展開後:', decompressed.byteLength, 'bytes');
ボタンをクリックすると実行されます

実験 7-E: ReadableStream → Response → 各種型

🔰 初心者向け解説

StreamはデータをChunk（塊）単位で順番に流す仕組みです。「水道管（Stream）から水（データ）を受け取って、コップ（text）やバケツ（blob）に入れる」イメージです。

  • ReadableStream: データの出口（蛇口）
  • WritableStream: データの入口（排水口）
  • chunk: 一度に流れるデータの塊（水の流れ）
  • new Response(stream): StreamをResponseに包んで .text()/.blob()等で取り出せるようにする

Step 1: ReadableStreamの手動読み取り

STEP 1 — getReader() でchunkを1つずつ読み取る
// ReadableStreamを自分で作る
const stream = new ReadableStream({
  start(controller) {
    controller.enqueue(new TextEncoder().encode('chunk1: Hello '));
    controller.enqueue(new TextEncoder().encode('chunk2: World '));
    controller.enqueue(new TextEncoder().encode('chunk3: !'));
    controller.close();
  }
});

// reader でchunkを1つずつ読み取る
const reader = stream.getReader();
let chunks = [];
while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  chunks.push(value); // Uint8Array
  console.log('chunk受信:', new TextDecoder().decode(value));
}

// 全chunkを結合して文字列に
const total = new Uint8Array(chunks.reduce((a, c) => a + c.byteLength, 0));
let offset = 0;
for (const chunk of chunks) { total.set(chunk, offset); offset += chunk.byteLength; }
console.log(new TextDecoder().decode(total));
ボタンをクリックすると実行されます

Step 2: new Response(stream) で各種型に変換

STEP 2 — StreamをResponseに包んで .text() / .blob() / .arrayBuffer() で取得
// Streamを作成
function makeStream(text) {
  const bytes = new TextEncoder().encode(text);
  return new ReadableStream({
    start(controller) {
      // 10バイトずつ流す
      for (let i = 0; i < bytes.length; i += 10) {
        controller.enqueue(bytes.slice(i, i + 10));
      }
      controller.close();
    }
  });
}

// new Response(stream) でラップして各種型に変換
const asText = await new Response(makeStream('Hello!')).text();
const asBlob = await new Response(makeStream('Hello!')).blob();
const asAB   = await new Response(makeStream('Hello!')).arrayBuffer();
ボタンをクリックすると実行されます

Step 3: fetchレスポンスのbodyはReadableStream

STEP 3 — response.body がReadableStreamであることを確認
// fetchレスポンスのbodyプロパティ
const res = await fetch('https://httpbin.org/get');
console.log(res.body instanceof ReadableStream); // true
console.log(res.bodyUsed);                       // false（まだ読んでいない）

// .text() を呼ぶと内部でStreamを消費する

console.log(res.bodyUsed); // true（読み終わり）
ボタンをクリックすると実行されます
response.body は ReadableStream です。.text()/.json()/.blob()/.arrayBuffer()/.formData() はすべてこのStreamを消費する便利メソッドです。大きなファイルの場合は .body.getReader() でchunkずつ処理することで、メモリ効率よく扱えます。
✅ 確認チェックリスト
ReadableStream を getReader() でchunkずつ読み取れた
new Response(stream) で .text()/.blob()/.arrayBuffer() に変換できた
response.body が ReadableStream であることを確認した
