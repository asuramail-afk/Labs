00

async関数の戻り値はPromise

基本

async function は常に Promise を返す。 return 42 と書いても Promise.resolve(42) と同じ。 まずこの事実を instanceof で確認する。

🔰 シンタックスシュガーとは？
「構文糖衣」＝ 機能は同じだが、読み書きしやすくした省略記法。
async/await は Promise を「同期コード風に」書ける糖衣構文。内部では Promise そのもの。 
  async/await          Promise.then()
  ┌──────────┐         ┌──────────────┐
  │ 見た目が  │  ===    │ 実際に動く   │
  │ 同期的    │  変換   │ 非同期処理   │
  └──────────┘         └──────────────┘
async/await版
async function greet() {
  return "Hello";
}

const result = greet();
console.log(result instanceof Promise); // true
console.log(result);  // Promise {<fulfilled>: "Hello"}
Promise版（等価）
function greetP() {
  return Promise.resolve("Hello");
}

const resultP = greetP();
console.log(resultP instanceof Promise); // true
console.log(resultP); // Promise {<fulfilled>: "Hello"}
01

awaitはthen()の書き換え

基本

await expr は expr.then(result => ...) と同じ。 両方を実行して出力が一致することを確認する。

async/await版
async function fetchData() {
  const data = await Promise.resolve(100);
  console.log("[await]", data);
}
fetchData();
Promise版（等価）
function fetchDataP() {
  return Promise.resolve(100)
    .then(data => {
      console.log("[then]", data);
    });
}
fetchDataP();
02

逐次処理の等価変換

基本

複数の await を連続で書く ＝ .then().then() のチェーン。 各ステップの実行順序と値の受け渡しが同一であることを確認する。

async/await版
async function pipeline() {
  const a = await Promise.resolve(1);
  const b = await Promise.resolve(a + 10);
  const c = await Promise.resolve(b * 2);
  return c; // 22
}
Promise版（等価）
function pipelineP() {
  return Promise.resolve(1)
    .then(a => Promise.resolve(a + 10))
    .then(b => Promise.resolve(b * 2))
    .then(c => c); // 22
}
03

try/catchとcatch()の同一性

中級

async関数内の try/catch は .catch() に変換される。 同じエラーを両方式でキャッチし、エラーメッセージが一致することを確認する。

async/await版
async function risky() {
  try {
    await Promise.reject(
      new Error("Boom!")
    );
  } catch (e) {
    return e.message;
  }
}
Promise版（等価）
function riskyP() {
  return Promise.reject(
    new Error("Boom!")
  ).catch(e => {
    return e.message;
  });
}
04

並列実行 Promise.all

中級

await を連続で書くと逐次実行になる。 並列にするには Promise.all を使う必要があり、ここは糖衣構文ではカバーしきれない領域。 逐次 vs 並列の実行時間差を計測する。

❌ 逐次（遅い）
async function sequential() {
  const a = await delay(500);
  const b = await delay(500);
  return [a, b]; // ~1000ms
}
✅ 並列（速い）
async function parallel() {
  const [a, b] = await Promise.all([
    delay(500),
    delay(500)
  ]);
  return [a, b]; // ~500ms
}
05

戻り値のPromiseチェーン

中級

async関数の戻り値に対して .then() を繋げられる。 これは async 関数が Promise を返す証拠であり、Promise チェーンとの相互運用性を示す。

async 戻り値に .then() を接続
async function getNum() {
  return 5;
}

// async関数の戻り値 = Promise なので .then() が使える
getNum()
  .then(n => n * 3)
  .then(n => n + 1)
  .then(n => console.log("結果:", n)); // 16

// 逆に .then() チェーンを await で受けることもできる
async function mixed() {
  const val = await Promise.resolve(5)
    .then(n => n * 3)
    .then(n => n + 1);
  console.log("mixed結果:", val); // 16
}

Babelトランスパイル結果

上級

Babelは async/await をトランスパイルすると Generator + Promise に変換する。 この変換結果は「async/awaitが本質的にPromiseである」最も直接的な証拠。

変換前 (ES2017)
async function fetchUser(id) {
  const res = await fetch(`/api/users/${id}`);
  const user = await res.json();
  return user.name;
}
変換後 (ES5 相当・簡略版)
function fetchUser(id) {
  return _asyncToGenerator(
    function* () {
      const res = yield fetch(
        `/api/users/${id}`
      );
      const user = yield res.json();
      return user.name;
    }
  )();
}

// _asyncToGenerator は Generator を
// Promise チェーンに変換するヘルパー
🔰 _asyncToGenerator の中身（概念）
  async function f() {        function f() {
    const a = await p1;  →      return new Promise((resolve, reject) => {
    const b = await p2;            p1.then(a => {
    return b;                        p2.then(b => {
  }                                    resolve(b);
                                     }).catch(reject);
                                   }).catch(reject);
                                 });
                               }
await ごとに .then() のネストが1段深くなる。
async/await はこの「ネスト地獄」を平坦に書くための糖衣構文。
07

マイクロタスクの実行順

上級

await と .then() は同じマイクロタスクキューを使う。 両者を混在させた時の実行順序が同一であることを観察する。

実行順序テスト
// テストA: async/await
async function testA() {
  console.log("A1: 関数開始");
  await Promise.resolve();
  console.log("A2: await後");
}

// テストB: Promise.then (等価)
function testB() {
  console.log("B1: 関数開始");
  return Promise.resolve().then(() => {
    console.log("B2: then後");
  });
}

console.log("1: 開始");
testA();
console.log("2: testA呼出後");
testB();
console.log("3: testB呼出後");
// 出力順: 1 → A1 → 2 → B1 → 3 → A2 → B2

async無しでawait相当を実装

上級

async/await を一切使わず、同じ処理を Promise だけで書き直す。 「完全に同じ動作になる」ことが「糖衣構文である」最終的な証明。

async/await版
async function workflow() {
  try {
    const userId = await getUser();
    const posts = await getPosts(userId);
    const first = posts[0];
    const comments = await getComments(first);
    return { userId, first, comments };
  } catch (e) {
    return { error: e.message };
  }
}
Promise版（完全等価）
function workflowP() {
  return getUser()
    .then(userId => {
      return getPosts(userId)
        .then(posts => {
          const first = posts[0];
          return getComments(first)
            .then(comments =>
              ({ userId, first, comments })
            );
        });
    })
    .catch(e =>
      ({ error: e.message })
    );
}
観点async/awaitPromise.then
可読性 ✅ 同期コード風 ❌ ネストが深い
エラー処理 ✅ try/catch ⚠️ .catch() の位置注意
デバッグ ✅ スタックトレース明確 ❌ 匿名関数の連鎖
実行結果 完全に同一
